Дата: 2026-05-22  
Контекст: адаптация практикума «Смыслокодинг» под Obsidian + Cursor  
Цель: сделать Cursor не просто редактором кода, а операционным партнёром Антона для второго мозга, бизнеса, CRM, MVP и high-ticket консалтинга.

---

## 1. Что было задачей

Нужно было взять материалы практикума «Смыслокодинг» и адаптировать их под твою реальную среду:

- не Claude Code + VS Code как в обучении;
- а Cursor + Obsidian vault;
- с учётом твоего фокуса: недвижимость, новостройки, CRM, n8n, AI-автоматизация, консалтинг;
- с фильтром цели: 500k+ RUB/мес;
- с принципами: путь наименьшего сопротивления, win-win, снижение важности, система вместо героизма.

Главная идея внедрения:

> Не копировать структуру курса один в один, а встроить методологию в уже существующий Obsidian-vault и `.cursor/`.

## 3. Главный вывод по практикуму

Практикум не про «научиться кодить».

Он про систему:

```text
смысл → контекст → план → фаза → критика → результат → правило в базу
```

Для тебя это переводится так:

```text
экспертиза в недвижимости → Obsidian-контекст → Cursor → MVP / КП / CRM-аудит → продажа / пилот
```

То есть смыслокодинг в твоём случае должен работать не как игрушка разработчика, а как механизм упаковки твоей экспертизы в:

- CRM-диагностики;
- офферы для застройщиков;
- партнёрские продукты;
- обучающие материалы;
- AI-автоматизации;
- демо-продукты и MVP;
- контент для личного бренда.

## 4. Как адаптирован Claude Code под Cursor

В курсе используется структура Claude Code. В твоей системе она адаптирована так:

| В курсе | В твоём Cursor-vault |
|--------|-----------------------|
| `CLAUDE.md` | `.cursor/rules/` + `.cursor/README.md` + проектные README |
| `.claude/skills/` | `.cursor/skills/` |
| `ai-clone/` | `Антон 2.0/Антон 2.0 — профиль обо мне.md` + `Антон 2.0/ИИ/ИИ-пульт.md` |
| `business/` | `Продукты/`, `Застройщики/`, `ЭКСПЕРТ СИТИ/`, `Стратегия первички/`, CRM-досье |
| `mastery/` | `Практикум Смыслокодинг/Методология в Cursor — эссе.md` + методички |
| `plans/` | `Антон 2.0/Планы/` |
| `retrospectives/` | `Антон 2.0/retrospectives/` |
| Bypass Permissions | Plan Mode + подтверждение важных изменений |
| Claude Chrome | При необходимости: browser automation / ручной промт для браузерного агента |

Важно:

> Cursor уже является твоей рабочей средой. Не нужно ломать систему и переезжать в Claude Code только потому, что так было в обучении.

## 8. Что принципиально НЕ делали

Сознательно не делалось:

- не переносили весь курс в новую структуру `business/`, `ai-clone/`, `mastery/` один в один;
- не создавали отдельный `CLAUDE.md`, потому что в Cursor сильнее работают `.cursor/rules` + README + Obsidian-карта;
- не трогали `.obsidian/`;
- не редактировали план-файл из `.cursor/plans/`, потому что пользователь запретил;
- не запускали деплой;
- не подключали n8n;
- не лезли в API / базы / ПДн;
- не делали полноценный SaaS.

Причина:

> 80/20: сначала карта, метод, правила, ритуалы и один денежный MVP. Потом автоматизация.

## 12. Главное решение

Система теперь построена вокруг принципа:

> Cursor должен не просто выполнять задачи, а помогать выбирать правильные задачи.

Для твоей цели 500k+ RUB/мес это значит:

- меньше «интересных» AI-экспериментов;
- больше MVP, которые можно показать агентству / застройщику;
- меньше важности и перфекционизма;
- больше быстрых демо, созвонов, пилотов;
- автоматизация только после подтверждения спроса.
