# Инструкция: Как обработать большие транскрипции без таймаутов

## Проблема

При обработке больших файлов (1000+ строк) в Cursor возникает ошибка:
```
Tool call ended before result was received
ERROR_USER_ABORTED_REQUEST
```

Это происходит потому, что обработка занимает слишком много времени и превышает лимит таймаута.

---

## ✅ Решение: Обработка по частям

Есть **3 способа** решить эту проблему:

---

## 🚀 Способ 1: Автоматическая разбивка (РЕКОМЕНДУЕТСЯ)

### Шаг 1: Разбей файл на части

Используй скрипт `split_large_transcription.py`:

```bash
cd "Антон 2.0/ИИ/Скрипты"
python split_large_transcription.py "../../Встреча по ИИ с Кожевниковой.md" --chunk-size 300
```

**Параметры:**
- `--chunk-size 300` — размер части в строках (можно изменить: 200, 250, 400)
- `--output-dir` — директория для сохранения (опционально)

### Шаг 2: Обработай каждую часть

1. Открой созданный файл `*_ИНДЕКС_ЧАСТЕЙ.md`
2. Обработай **каждую часть по очереди**, начиная с части 1

**Запрос для каждой части:**
```
Обработай эту часть транскрипции используя промпт @Промпт - Обработка больших транскрипций (по частям).md

Это часть 1 из 4, строки 1-300.
```

**Важно:** Обрабатывай части **последовательно**, не пропуская.

### Шаг 3: Объедини результаты

После обработки всех частей:

```
Объедини все обработанные части в финальный документ используя промпт @Промпт - Обработка больших транскрипций (по частям).md

[вставь все обработанные части из предыдущих шагов]
```

---

## ✂️ Способ 2: Ручная разбивка

Если не хочешь использовать скрипт:

### Шаг 1: Разбей файл вручную

1. Открой исходный файл
2. Раздели его на части по 200-400 строк
3. Сохрани каждую часть в отдельный файл:
   - `Встреча_часть_1.md`
   - `Встреча_часть_2.md`
   - и т.д.

### Шаг 2-3: Аналогично способу 1

---

## 🎯 Способ 3: Упрощённая обработка

Если нужна только структуризация без полного анализа:

### Используй упрощённый запрос:

```
Структурируй этот файл транскрипции по темам. 

Разбей на логические блоки с заголовками, но НЕ создавай:
- Резюме
- Ключевые мысли
- Задачи
- Анализ

Только структуризация по темам.
```

Это быстрее и не вызовет таймаут.

---

## 📊 Рекомендуемые размеры частей

| Размер файла | Размер части | Количество частей |
|--------------|--------------|-------------------|
| 500-800 строк | 300 строк | 2-3 части |
| 800-1500 строк | 300 строк | 3-5 частей |
| 1500+ строк | 250 строк | 6+ частей |

**Совет:** Если всё равно возникают таймауты, уменьши `--chunk-size` до 200-250 строк.

---

## 🔧 Устранение проблем

### Проблема: Скрипт не запускается

**Решение:**
```bash
# Убедись, что Python установлен
python3 --version

# Если нет, установи Python 3.8+
```

### Проблема: Всё равно таймаут на одной части

**Решение:**
1. Уменьши размер части до 200 строк
2. Или используй способ 3 (упрощённая обработка)

### Проблема: Потеря контекста между частями

**Решение:**
- При обработке каждой части указывай: "Это часть X из Y"
- При финальной обработке вставляй ВСЕ обработанные части целиком

---

## 💡 Советы

1. **Начни с малого** — сначала попробуй обработать одну часть, чтобы понять скорость
2. **Сохраняй результаты** — после обработки каждой части сохраняй результат
3. **Используй индексный файл** — он поможет не потеряться в частях
4. **Проверяй качество** — после финальной обработки проверь, что ничего не потеряно

---

## 📝 Пример полного процесса

```bash
# 1. Разбивка
python split_large_transcription.py "Встреча по ИИ с Кожевниковой.md"

# 2. В Cursor обрабатываешь каждую часть:
#    - Часть 1 из 4
#    - Часть 2 из 4
#    - Часть 3 из 4
#    - Часть 4 из 4

# 3. Объединяешь все обработанные части в финальный документ
```
