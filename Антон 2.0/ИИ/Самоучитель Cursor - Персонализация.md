# Самоучитель Cursor — Персонализация под себя

## Введение: Почему нужно персонализировать Cursor

Cursor — это AI-ассистент для программирования, который работает прямо в вашем редакторе кода. Без персонализации он отвечает «как все», не зная вашего контекста, стиля работы и сферы деятельности.

**После персонализации Cursor будет:**
- Знать, кто вы и чем занимаетесь
- Понимать ваш стиль работы и предпочтения
- Адаптировать ответы под ваши направления (недвижимость, консалтинг, технологии)
- Следовать вашим принципам (SOLID, KISS, практичность)
- Давать структурированные, практичные ответы без воды
- Предлагать решения с учетом монетизации и бизнес-ценности

**Результат:** Cursor превращается из «обычного ассистента» в «личного эксперта», который знает ваш бизнес и помогает принимать решения быстрее и эффективнее.

---

## Глава 1: Что такое Cursor и как он работает

### Что такое Cursor

Cursor — это редактор кода на базе VS Code, который использует AI для помощи в программировании. Он может:
- Генерировать код по описанию
- Объяснять существующий код
- Рефакторить и улучшать код
- Отвечать на вопросы о проектах
- Помогать с отладкой

### Как Cursor использует контекст

Cursor анализирует:
1. **Файл `.cursorrules`** — ваши персональные правила и инструкции
2. **Открытые файлы** — контекст текущей работы
3. **Структуру проекта** — понимание архитектуры
4. **Историю чата** — контекст разговора

**Ключевой момент:** Файл `.cursorrules` в корне проекта — это ваше «досье» для AI. Чем подробнее вы его заполните, тем лучше Cursor будет понимать вас.

## Глава 2: Создание файла `.cursorrules`

### Где размещать файл

**Вариант 1: Глобально для всех проектов**
- Создайте файл `.cursorrules` в домашней папке (`~/.cursorrules`)
- Применится ко всем проектам по умолчанию

**Вариант 2: Для конкретного проекта**
- Создайте файл `.cursorrules` в корне проекта
- Будет работать только для этого проекта

**Рекомендация:** Используйте глобальный файл для общих предпочтений и добавляйте проектные правила при необходимости.

### Структура файла `.cursorrules`

Файл — это обычный текстовый файл без расширения. Можете использовать Markdown для структурирования.

**Базовая структура:**
```
# Персональные правила для Cursor

## Кто я
[Ваше описание]

## Чем занимаюсь
[Ваши направления]

## Стиль работы
[Ваши предпочтения]

## Принципы
[Ваши принципы кодирования]

## Контекст бизнеса
[Сфера деятельности]
```

## Глава 5: Как создать и разместить файл `.cursorrules`

### Метод 1: Через терминал (Mac/Linux)

**Шаг 1: Откройте терминал**

**Шаг 2: Создайте файл в домашней папке**
```bash
cd ~
nano .cursorrules
```

**Шаг 3: Вставьте содержимое**
- Скопируйте текст из главы 4
- Вставьте в редактор (в nano: правый клик или Cmd+V)

**Шаг 4: Сохраните и выйдите**
- В nano: нажмите `Ctrl+O` (сохранить), затем `Enter`, затем `Ctrl+X` (выход)

### Метод 2: Через Cursor (рекомендуется)

**Шаг 1: Откройте Cursor**

**Шаг 2: Создайте файл `.cursorrules` в корне проекта**
1. В файловом дереве нажмите правой кнопкой на корень проекта
2. Выберите "New File"
3. Назовите файл `.cursorrules` (с точкой в начале!)

**Шаг 3: Вставьте содержимое**
- Скопируйте текст из главы 4
- Вставьте в файл

**Шаг 4: Сохраните**
- Cmd+S (Mac) или Ctrl+S (Windows/Linux)

### Метод 3: Глобальный файл через Finder/Проводник

**Для Mac:**
1. Откройте Finder
2. Нажмите `Cmd+Shift+G` (перейти к папке)
3. Введите `~` (тильда — домашняя папка)
4. Нажмите Enter
5. Создайте файл `.cursorrules` (с точкой!)
6. Откройте в любом текстовом редакторе и вставьте содержимое

**Важно:** Файлы, начинающиеся с точки (`.`), скрыты по умолчанию. Чтобы увидеть их в Finder, нажмите `Cmd+Shift+.` (точка).

### Проверка работы

**Как проверить, что файл работает:**

1. Откройте Cursor в любом проекте
2. Откройте чат (Cmd+L или Ctrl+L)
3. Задайте вопрос: "Кто я и чем занимаюсь?"
4. Cursor должен ответить, используя информацию из `.cursorrules`

Если ответ содержит вашу информацию — всё работает!

## Глава 6: Настройка через Settings Cursor

Помимо `.cursorrules`, в Cursor есть дополнительные настройки.

### Где находятся настройки

1. Откройте Cursor
2. Нажмите `Cmd+,` (Mac) или `Ctrl+,` (Windows/Linux)
3. Или: `Cursor → Settings` (в меню)

### Полезные настройки

**AI Settings (Настройки AI):**

1. **Codebase Indexing (Индексация кодовой базы)**
   - Включите, чтобы Cursor знал структуру проекта
   - Помогает давать более точные ответы

2. **Inline Edit (Встроенное редактирование)**
   - Настройте, как Cursor предлагает изменения
   - Рекомендуется: показывать изменения сразу

3. **Chat Model (Модель для чата)**
   - Выберите модель (GPT-4, Claude и т.д.)
   - GPT-4 обычно лучше для сложных задач

**Editor Settings (Настройки редактора):**

- Включите автодополнение
- Настройте форматирование кода
- Включите проверку ошибок (linting)

### Настройка правил для конкретного проекта

Если у вас несколько проектов с разными требованиями:

1. Создайте `.cursorrules` в корне конкретного проекта
2. Он переопределит глобальный файл для этого проекта

**Пример для проекта с Python:**
```
## Специфика проекта

### Язык и фреймворк
- Python 3.11+
- Flask/FastAPI для веб-API
- Используй type hints везде

### Стандарты кода
- PEP 8
- Максимальная длина строки: 100 символов
- Используй black для форматирования
```

## Глава 11: Решение типичных проблем

### Проблема 1: Cursor не использует `.cursorrules`

**Симптомы:**
- Cursor не знает о вас и вашем контексте
- Ответы общие, не персонализированные

**Решение:**
1. Проверьте, что файл называется именно `.cursorrules` (с точкой!)
2. Убедитесь, что файл в корне проекта или в `~/.cursorrules`
3. Перезапустите Cursor
4. Проверьте: задайте вопрос "Кто я и чем занимаюсь?"

### Проблема 2: Ответы слишком длинные

**Симптомы:**
- Cursor пишет много текста без пользы

**Решение:**
Добавьте в `.cursorrules`:
```
## О длине ответов
- Будь краток, но информативен
- Если нужно много деталей — используй списки
- Избегай повторений
```

### Проблема 3: Не учитывает контекст бизнеса

**Симптомы:**
- Предлагает решения, не подходящие для вашей сферы

**Решение:**
1. Добавьте больше контекста о бизнесе в `.cursorrules`
2. Укажите типичные задачи и сценарии
3. Приведите примеры решений, которые вам подходят

### Проблема 4: Не следует принципам (SOLID, KISS)

**Симптомы:**
- Генерирует код, который не соответствует вашим принципам

**Решение:**
1. Выделите принципы в отдельный раздел в `.cursorrules`
2. Попросите Cursor объяснить, как он следовал принципам
3. Уточните, если нужно: "Примени SOLID принципы к этому коду"
