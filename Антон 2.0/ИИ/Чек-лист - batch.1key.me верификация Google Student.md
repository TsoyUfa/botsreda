# Чек-лист: верификация Google Student через batch.1key.me

Пошаговая инструкция по использованию [batch.1key.me](https://batch.1key.me/) для пакетной верификации студенческого статуса Google (Google Student).

---

## Важно перед началом

- Сервис работает только с **verification ID**, полученными из процесса верификации Google (например, при переходе по ссылке из поддерживаемой страны).
- Россия **не входит** в список стран официального студенческого оффера Google; для получения verification ID обычно нужен доступ из поддерживаемой страны (VPN и т.п.).
- Чек-лист не заменяет официальную форму Google/SheerID, а помогает пройти уже созданную верификацию пакетно.

---

## Шаг 1. Капча или API-ключ

**Цель:** разблокировать возможность запуска верификации.

- [ ] **Вариант A.** Пройти **обе** капчи на странице batch.1key.me  
  - На странице два блока капчи (обычно reCAPTCHA или аналог).  
  - Оба должны быть отмечены как пройденные до нажатия «Start Verification».

- [ ] **Вариант B.** Задать API-ключ  
  - Нажать **«Set API Key»**.  
  - Ввести ключ сервиса, поддерживающего распознавание капчи (если вы его используете).  
  - После сохранения ключа капчи на странице могут не требоваться для запуска.

**Частая ошибка:** нажать «Start Verification», не пройдя капчи и не задав API-ключ — в этом случае верификация не запускается или сразу падает.

---

## Шаг 2. Ввод verification ID или URL

**Цель:** передать сервису идентификаторы сессий верификации Google.

- [ ] Открыть поле **«Input»** (или «Paste URLs directly»).

- [ ] Вставить один из форматов:
  - **Целые URL** — ссылки вида  
    `https://one.google.com/...?requestId=XXXXX...` или аналог.  
    Сервис сам извлечёт verification ID из параметров URL.
  - **Только verification ID** — строки вида `requestId` / идентификатор верификации, скопированные из URL или из ответа при открытии формы верификации Google.

- [ ] Проверить, что в блоке «Input» отображается что-то вроде «N IDs entered» (N > 0), а не «0 IDs entered».

**Где взять ID/URL:**
- При переходе по [one.google.com/verify/student](https://one.google.com/verify/student) или [gemini.google/students](https://gemini.google/students) в адресной строке или в редиректах могут появляться параметры с идентификатором сессии верификации.
- Скопировать полный URL или значение параметра (часто `requestId`, `verificationId` или похожее) и вставить в поле ввода.

**Частая ошибка:** вставлять логин/пароль от Google или e-mail — сервис ждёт именно **verification ID** или **URL с верификацией**, а не учётные данные.

---

## Шаг 3. Запуск верификации

- [ ] Убедиться, что капчи пройдены или задан API-ключ (шаг 1).

- [ ] Убедиться, что в поле ввода есть хотя бы один ID или URL (шаг 2).

- [ ] Нажать **«Start Verification»** (или «I'm Feeling Lucky» — в зависимости от интерфейса).

- [ ] Дождаться обработки. Результаты появятся в блоке **«Results»**: Success / Failed по каждому ID.

---

## Шаг 4. Результаты и экспорт

- [ ] В разделе **«Results»** посмотреть счётчики: Success, Failed, Total.

- [ ] При необходимости нажать **«Export»**, чтобы сохранить отчёт.

- [ ] При множественных неудачах проверить:
  - снова ли пройдены обе капчи / актуален ли API-ключ;
  - корректность и актуальность verification ID (не истекла ли сессия верификации Google);
  - что аккаунт/регион в принципе допускает студенческий оффер (см. список поддерживаемых стран в заметке «Страны — Google Gemini One студенческий оффер»).

---

## Краткая последовательность

1. Капча (обе) **или** Set API Key.  
2. Ввод ID/URL в Input → проверка «N IDs entered».  
3. Start Verification.  
4. Результаты в Results, при необходимости Export.
