# 🤖 Главный B2B Telegram-Бот Антона Цоя

Это единый Telegram-бот, спроектированный **исключительно для работы с риелторами и агентами по недвижимости**. Бот решает задачи автоматической выдачи лид-магнитов из Reels, проверки подписки на экспертный Telegram-канал, вовлечения через интерактивный тест и организации личного разбора звонков через двухсторонний чат поддержки.

---

## 🌟 Основной функционал

1. **Автоматическая воронка Reels:** 
   * Распознает 8 кодовых слов (`РЫНОК`, `LTV`, `ЭТАПЫ`, `ПРАВИЛА`, `РАСЧЕТ`, `ПУШКА`, `ЭКСПЕРТ`, `КОНТАКТ`) или реферальные ссылки глубокого старта (`/start reels_1`, `/start reels_ltv` и т.д.).
   * Перед отправкой файла запрашивает и автоматически проверяет подписку на Telegram-канал.
2. **Фоновый автодогрев (Drip Campaign):** 
   * Спустя 24 часа после скачивания лид-магнита бот отправляет ненавязчивое напоминание с предложением пройти тест.
3. **Интерактивный Экспресс-тест:** 
   * 10 профессиональных вопросов о продажах новостроек с моментальным показом правильных ответов и обоснований.
   * Начисление баллов и закрытие на спецпредложение (бесплатный аудит аудиозвонка).
4. **Двусторонний Чат Поддержки:** 
   * Позволяет агентам присылать вопросы и записи звонков. Сообщения перенаправляются в чат Антону.
   * Антон отвечает простым действием **Reply (Ответить)** на пересланное сообщение, и ответ доставляется агенту прямо от лица бота.
5. **Панель Администратора:** 
   * Выгрузка базы агентов с баллами за тест в CSV файл (`/leads`).
   * Массовая рассылка сообщений по сегментам базы (`/broadcast`).
   * Мгновенный сбор статистики по популярности кодовых слов и активности (`/stats`).

## 💻 Локальный запуск и разработка

### 1. Установите Python
Убедитесь, что у вас установлен Python версии **3.9 или выше**.

### 2. Клонируйте код и подготовьте переменные
В папке бота переименуйте `.env.example` в `.env` и впишите свои токен бота и ID:
```bash
cp .env.example .env
```

### 3. Создайте виртуальное окружение и установите зависимости
```bash
# Создание виртуального окружения
python3 -m venv venv

# Активация виртуального окружения
# На macOS / Linux:
source venv/bin/activate
# На Windows:
venv\Scripts\activate

# Установка библиотек
pip install -r requirements.txt
```

### 4. Запустите бота
```bash
python3 bot.py
```

## ☁️ Развертывание на Linux-сервере (Ubuntu/Debian)

Для бесперебойной работы 24/7 настройте бота как Systemd-сервис.

### 1. Загрузите файлы бота на сервер
С помощью Git или SFTP поместите папку проекта в директорию на сервере, например, в `/home/ubuntu/master_bot`.

### 2. Повторите настройку окружения на сервере
```bash
cd /home/ubuntu/master_bot
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
nano .env # Введите токен вашего продакшн-бота и ваш ID
```

### 3. Создайте Systemd Service-файл
Создайте конфигурационный файл демона:
```bash
sudo nano /etc/systemd/system/master_bot.service
```

Вставьте в него следующую конфигурацию (замените `ubuntu` на имя вашего пользователя и проверьте пути к проекту):

```ini
[Unit]
Description=General B2B Telegram Bot for Anton Tsoy
After=network.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu/master_bot
ExecStart=/home/ubuntu/master_bot/venv/bin/python bot.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

### 4. Запустите службу бота
Выполните команды для активации и запуска:
```bash
# Обновление конфигурации Systemd
sudo systemctl daemon-reload

# Включение автозапуска бота при старте сервера
sudo systemctl enable master_bot

# Запуск бота
sudo systemctl start master_bot

# Проверка статуса (убедитесь, что статус "active (running)")
sudo systemctl status master_bot
```

### 5. Полезные команды управления на сервере:
* **Перезапустить бота** (после изменения кода): `sudo systemctl restart master_bot`
* **Остановить бота:** `sudo systemctl stop master_bot`
* **Просмотр логов в реальном времени:** `journalctl -u master_bot -f`
