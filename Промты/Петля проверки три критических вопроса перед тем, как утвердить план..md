КОГДА ПРИМЕНЯТЬ

После того как агент написал план реализации, ДО начала кодинга. Каждый раз, когда план кажется тебе «вроде ок» - прогони через эти 3 вопроса.

ЗАЧЕМ НУЖЕН

Это финальный фильтр перед тем, как сжечь часы на реализацию плохого плана. Один раз отловить дырку в плане - сэкономить день рефакторинга.


Перед тем как финализировать план, ответь на 3 вопроса:

1. РЕШАЕТ ЛИ ЭТО ПРОБЛЕМУ?
   Сопоставь каждый пункт плана с acceptance criteria из спеки.
   Если хоть один критерий не покрыт - план неполный.

2. ЭТО САМОЕ ЭФФЕКТИВНОЕ РЕШЕНИЕ?
   Найди: кто уже решал эту проблему? Какой подход выбрали?
   Назови 2-3 альтернативных подхода (включая найденные через research).
   Для каждого: плюсы, минусы, трудоёмкость.
   Обоснуй, почему выбранный подход лучше всех альтернатив.

3. ЕСТЬ ЛИ В ПЛАНЕ «КОД РАДИ КОДА»?
   Каждое изменение должно напрямую обслуживать acceptance criteria.
   Если изменение не привязано к задаче - удали.
   Drive-by рефакторинг - отдельная задача, не часть этой.

Только после прохождения всех 3 вопросов план можно считать готовым.
