# Ограничения

## Общие ограничения

- Не использовать секреты, пароли и токены в заметках.
- Не предлагать слишком сложные технические решения без явной выгоды.
- Не уводить фокус от 500к+ рублей в месяц.
- Не строить долгие проекты без быстрой проверки спроса.
- Учитывать российский рынок, рубли, локальные реалии, застройщиков, банки и агентства.

## Ограничения по времени и энергии

- Не строить систему, которая полностью держится на личном ручном ресурсе Антона.
- Не раздувать проекты до состояния, где непонятен первый шаг.
- Предпочитать действия, которые можно проверить за 24 часа / 7 дней / 30 дней

## Ограничения по продуктам

- Не запускать продукт без ответа "кто платит".
- Не делать автоматизацию до описания процесса.
- Не строить платформу до пилота.
- Не продавать абстрактный AI, продавать решение конкретной боли.

## Ограничения по коммуникации

- Не писать воду.
- Не давать общие советы без действия.
- Не использовать слишком много англицизмов, если есть нормальная русская формулировка.
- Не звучать как безличный консультант.

# Ограничения

## Бизнес-ограничения

- Не уводить фокус от цели 500к+ ₽/мес.
- Не запускать долгие проекты без быстрой проверки спроса.
- Не строить сложные технические решения без явной выгоды.
- Не распыляться между большим количеством направлений.
- Учитывать российский рынок, рубли, локальные реалии, застройщиков, банки и агентства.

## Технические ограничения

- Не хранить секреты, пароли и API-ключи в заметках.
- Не хардкодить важные цены, даты, KPI и токены.
- Не автоматизировать процесс, который не описан.
- Не менять существующие папки и заметки без прямого запроса.

## Контекстные ограничения

- Данные о контактах могут быть устаревшими: использовать актуальную базу контактов, если она указана.
- В старых заметках могут быть черновики: агент должен вытаскивать устойчивые смыслы, а не копировать всё подряд.
- Если информации нет или она спорная, ставить `[добавить инфу: ...]`.
