# Cursor Workflow Coach (смыслокодинг)

## Опора

- `Практикум Смыслокодинг/Методология в Cursor — эссе.md`
- `Антон 2.0/Cursor — карта навигации.md`
- Rules: `delivery-to-result-flow`, `mvp-money-filter`, `security-pd-review`

## Когда включать

- Пользователь спрашивает «как по смыслокодингу», «с чего начать в Cursor»
- Новый продукт / MVP / демо для недвижимости
- Нужен план с фазами, а не сразу код
- Перенос из Claude Code / VS Code в Cursor

## Процесс (выдай по шагам)

### Шаг 0 — маршрут контекста

Укажи, какие 3–5 файлов прочитать первыми (профиль, бизнес-раздел, план).

### Шаг 1 — намерение

Сформулируй: кому, боль, результат за 7 дней, метрика успеха.

### Шаг 2 — research (если продукт)

Кто на рынке, цены, 3 паттерна украсть, 3 ошибки не повторять. Файл или раздел в плане.

### Шаг 3 — план

Создать/обновить `Антон 2.0/Планы/YYYY-MM-DD-название.md`:
- фазы с `[ ]`
- критерий «сделано»
- что не входит в MVP

### Шаг 4 — фильтры

- `mvp-money-filter`: GO/NO-GO
- «10 причин провала» — список + правки

### Шаг 5 — реализация

- Оркестратор: держит план, выдаёт промты фаз
- Исполнитель: одна фаза, чистый чат
- После фазы: security-pd-review + обновление плана

### Шаг 6 — закрытие

- Ссылка на демо / что показать клиенту
- Ретроспектива в `Антон 2.0/retrospectives/`
- Правило в skill/rule если была ошибка
