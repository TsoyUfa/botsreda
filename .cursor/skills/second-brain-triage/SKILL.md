# Second Brain Triage

## Модель 4 папок (смыслокодинг)

| Папка | Вопрос | Примеры |
|-------|--------|---------|
| **1. Бизнес** | Что продаём, кому, цифры, продукты? | КП, продукты, застройщики, CRM-стандарты, кейсы |
| **2. План** | Что делаем сейчас, сроки? | Инбокс, контент-план с датами, 90 дней по клиенту |
| **3. Мой клон** | Как думает и говорит Антон? | Профиль, ToV, трансерфинг, принципы, cursor-rules |
| **4. Мастерство** | Как делать хорошо (метод)? | Методички Reels, шаблоны, промты, конспекты практикума |

**Правило:** сырые материалы (ДЗ учеников, транскрипты, html-дампы) → `raw/` внутри нужной ветки, не в корень для агента.

## Входные данные

- Файл(ы), папка или список путей.
- Задача: разовая сортировка / массовая / «куда положить этот черновик».

## Процесс

1. Для каждого элемента определи **тип**: идея / операционка / контент / партнёрство / обучение / личное / техническое / дубликат.
2. Назначь **целевую папку** (1 из 4) + подпапку по смыслу.
3. Реши: **канонический файл** / **ссылка** / **raw** / **архив** / **удалить дубликат**.
4. Если заметка содержит действие — извлеки **один шаг на 24ч** (формат как в `/knowledge`).
5. Если элемент пересекается с `Антон 2.0/Инбокс задач — май 2026.md` — предложи ID или новую строку в инбокс.

## Формат ответа

### Сводка
| Файл/папка | Тип | Куда | Действие |
|------------|-----|------|----------|
| `путь` | … | Бизнес/… | переместить / оставить / raw / ссылка |

### Дубликаты и конфликты
- что объединить, что оставить каноническим.

### Извлечённые действия (если есть)
- `Контекст` / `Решение` / `Следующий шаг` / `Кому` / `Срок`
- метки: `Сегодня` | `На этой неделе` | `В бэклог`

### Рекомендации по vault
- 1–3 структурных улучшения (индекс, CLAUDE.md, недостающий `Индекс.md` в папке).

### Не трогать
- `.obsidian/`, `.cursor/` (кроме явного запроса), бинарники без оцифровки.

## Связь с правилом

При разборе одной заметки на **решение** — используй также `.cursor/rules/knowledge-execution-flow.mdc` или `/knowledge`.

## Антипаттерны

- Не предлагать перенос 50+ файлов без приоритета — сначала топ-10 по ROI.
- Не класть ДЗ учеников в «Мой клон».
- Не дублировать KPI в markdown — помечать «взять из БД/CRM при внедрении Live Matrix».
