## Входные данные (запроси, если нет)

- Кто клиент диагностики: агентство / застройщик / своя команда.
- CRM (если есть): Bitrix24 / amo / другое / нет CRM.
- Текущие этапы воронки (список или скрин).
- Метрики: лиды, встречи, брони, сделки за период (хотя бы порядок величин).
- Главная боль: «теряем на …» / «нет дисциплины» / «нет прозрачности».

## Процесс

1. **Собери эталонную воронку** для новостроек (7–10 этапов): от лида до сделки + пост-сервис.
2. **Сопоставь** с текущими этапами пользователя — найди пропуски и дубли.
3. **Найди 3 узких места** по конверсии или по поведению (нет статуса, нет follow-up, нет причины отказа).
4. **SLA-чеклист** (минимум):
   - первый ответ ≤ 15 мин;
   - следующий шаг всегда в CRM;
   - причина отказа обязательна;
   - follow-up с датой;
   - после сделки — отзыв/рекомендация.
5. **MVP на 7–14 дней**: не «внедрить Bitrix», а 3–5 правил, которые дадут +X% к конверсии.
6. **Win-win**: как прозрачность воронки помогает агентам, РОПу, застройщику.
