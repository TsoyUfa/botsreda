# 6. MVP-3 — Telegram-бот, генераторы, аналитика

## 6.1. Скоуп MVP-3

**Добавляем к MVP-2:**
- Telegram-бот для быстрых `+1` действий и push-уведомлений.
- Генератор PDF фиксации уникальности.
- Генератор PDF финмодели.
- Лента событий потока с автопостингом в общий Telegram-чат группы (опц.).
- Расширенный кабинет наставника: сводка по потоку.
- Базовая продуктовая аналитика (PostHog) и админка для просмотра.

**Вне скоупа MVP-3:**
- Интеграция с агрегаторами (Авито/Тренд).
- Командные челленджи между потоками.
- Mobile-нативные приложения.
- Видео-уроки внутри дашборда.

**Цель MVP-3:** снять трение «достать дашборд → залогиниться → ткнуть кнопку», вынеся самые частые действия в Telegram, и дать наставнику цифры для управления потоком.

## 6.2. Telegram-бот

### 6.2.1. Привязка аккаунта

Флоу:
1. Агент в `/profile` нажимает «Привязать Telegram».
2. Дашборд показывает уникальный код и ссылку на бота.
3. В боте `/start <код>` — бот связывает `telegram_user_id` с `user_id`.
4. После привязки — приветственное сообщение.

### 6.2.2. Команды бота

| Команда | Действие |
|---------|----------|
| `/start` | привязка / приветствие |
| `/today` | главная задача дня + счётчики |
| `/touch [клиент?] [комментарий]` | +1 касание |
| `/dialog [клиент?] [комментарий]` | +1 диалог |
| `/selection [клиент?]` | +1 подборка |
| `/show [клиент?]` | +1 показ |
| `/post <ссылка>` | +1 контент-единица |
| `/me` | мой ранг, XP, монетки, место в рейтинге |
| `/week` | прогресс по текущей неделе курса |
| `/help` | список команд |

### 6.2.3. Inline-клавиатура

После каждой команды `+1` бот показывает inline-кнопки:
- «Добавить ещё одно» (по типу).
- «Привязать к клиенту» — открывает inline-поиск клиентов.
- «Открыть в дашборде».

### 6.2.4. Push-уведомления через бота

| Событие | Сообщение |
|---------|-----------|
| Наставник принял ДЗ | «✅ Неделя 4 принята! +300 XP. Открыть» |
| Наставник вернул на доработку | «✏️ Неделя 4 на доработку: <фидбэк>» |
| Получена ачивка | «🏆 Получена ачивка «Закрыватель»! +200 XP» |
| Достигнут ранг | «🎖 Поздравляем — теперь вы Опытный» |
| Получен бонус | «💰 Бонус Стартер: +5 000 ₽ к ближайшей выплате» |
| Сделка подтверждена РОПом | «🎉 Сделка по ЖК <X> подтверждена. +1 монетка, +1 000 XP» |
| Streak неделя | «🔥 7 дней подряд! +100 XP за серию» |
| Сегодня — главная задача | в 08:00 утра, если есть «висящее» ДЗ |
| ДЗ просрочено | в 18:00, если статус `draft` за 24 ч до дедлайна |

> Эмодзи в сообщениях бота используются точечно как маркеры событий (по запросу пользователя — эмодзи только там, где они уместны и явно нужны).

### 6.2.5. Технические детали

- **Стек:** grammY (TypeScript) на Supabase Edge Function `/functions/telegram-webhook`.
- **Сессии:** Supabase KV или столбец `telegram_session jsonb` в `users`.
- **Антифрод:** те же правила, что в дашборде (дневные капы, минимум полей).
- **Локализация:** только русский.
- **Безопасность:** проверка `secret_token` Telegram для webhook.

## 6.3. Генератор PDF фиксации уникальности

### Сценарий

1. Агент открывает карточку клиента → секция «Фиксация уникальности».
2. Жмёт «Сформировать PDF».
3. PDF собирается на серверной стороне (`@react-pdf/renderer` или Puppeteer на Edge).
4. Открывается превью + кнопки «Скачать», «Отправить застройщику» (mailto или ссылка).

### Шаблон PDF

Полностью соответствует шаблону из курса (Неделя 4):
- Шапка: лого СРЕДА, дата.
- ФИО супруга/супруги, телефоны.
- ЖК.
- Агент: ФИО, телефон, email.
- Предпочтения по менеджеру.
- Дополнительная информация.
- Чек-боксы: «Заявка отправлена», «Уникальность подтверждена».
- Подвал: контакты агентства.

### Хранение

Сгенерированный PDF записывается в Supabase Storage в бакет `uniqueness-fixations/<user_id>/<fixation_id>.pdf` со ссылкой в `uniqueness_fixations.pdf_url`.

## 6.4. Генератор PDF финмодели

### Сценарий

Аналогично фиксации: открывается из карточки клиента или из секции `/program/5`.

### Шаблон

- Шапка: ФИО клиента, дата, агент.
- Запрос клиента (бюджет, состав семьи, цель покупки) — поля из диагностики.
- Таблица сравнения 2–3 вариантов (как в курсе): ЖК, тип квартиры, стоимость, ПВ, кредит, программа, ставка, срок, платёж, плюсы, минусы.
- Сравнение с арендой (если в диагностике указано, что клиент арендует).
- Сценарий рефинансирования (одно предложение).
- Подвал: «Финмодель носит ознакомительный характер, итоговые условия согласовываются с банком».

### Расчёт платежа

Используем стандартную формулу аннуитета. Логика — в `src/domain/mortgage.ts`:

```ts
export function annuityPayment(amount: number, ratePct: number, years: number): number {
  const monthlyRate = ratePct / 100 / 12
  const months = years * 12
  if (monthlyRate === 0) return amount / months
  const factor = Math.pow(1 + monthlyRate, months)
  return Math.round((amount * monthlyRate * factor) / (factor - 1))
}
```

Покрыто юнит-тестами на сценариях курса (платёж 25 800 при кредите 4 млн, 6%, 25 лет).

## 6.5. Лента событий и автопостинг в чат группы

### Внутренняя лента

Уже была в MVP-1 на Hub. В MVP-3 расширяем:
- Источники событий: XP, ранг, ачивка, монетка, сделка, фиксация уникальности (без личных данных).
- Фильтр: «весь поток / только мои события / только сделки и ачивки».
- История 30 дней.

### Автопостинг в Telegram-чат группы

- Только публичные события: ранг, ачивка, монетка.
- Никогда не постим: имена клиентов, суммы сделок, личные данные.
- Управляется фичефлагом `posting_enabled` per cohort.
- Формат: «Айгуль получила ачивку «Закрыватель»! 🏆»
- Если агент отключил «делиться достижениями» в `/profile/settings` — события не постятся.

## 6.6. Расширенный кабинет наставника

### Экраны

#### `/mentor/cohort/[id]`
- Сводная таблица по подопечным:
  - Имя, текущая неделя, % сдачи ДЗ, кол-во касаний за неделю, кол-во сделок, ранг, XP.
  - Светофор: зелёный (всё хорошо), жёлтый (риск), красный (критично).
- Фильтр «риск-зона»: те, у кого нет действий 5+ дней или ДЗ просрочены.

#### `/mentor/insights`
- Графики (PostHog Embeddable Insights):
  - DAU/WAU потока.
  - Кривая retention.
  - Воронка прохождения недель (где дроп-офф).
  - Среднее время от подачи ДЗ до приёма.

#### `/mentor/queue`
- Очередь проверок: только ДЗ в `review`, отсортированные по сроку подачи.
- Таймер у каждого: сколько часов висит.

## 6.7. Аналитика

### События в PostHog

| Событие | Свойства |
|---------|----------|
| `user_signed_up` | cohort, role |
| `homework_submitted` | week, homework_code, attachments_count |
| `homework_accepted` | week, homework_code, time_to_review_h |
| `action_logged` | type, has_client, has_comment, source (web|tg) |
| `client_created` | source |
| `deal_paid` | jk |
| `coin_earned` | total_coins |
| `bonus_unlocked` | bonus_code |
| `rank_up` | from, to |
| `achievement_granted` | code |
| `tg_command_used` | command |

### Базовые дашборды (PostHog)

1. **Активация** — % новичков, сделавших ≥1 действие в первые 24 ч.
2. **Прохождение программы** — воронка по неделям.
3. **Использование Telegram-бота** — % действий, прилетевших из бота.
4. **Время приёмки ДЗ** — среднее, p50, p90.
5. **Распределение по рангам** в когорте.

## 6.8. Acceptance criteria MVP-3

- [ ] Telegram-бот привязывается к аккаунту, корректно выполняет 9 команд.
- [ ] Все push-уведомления доставляются (вебхуки + ретраи при ошибках).
- [ ] PDF фиксации уникальности генерируется и сохраняется в Storage, ссылка отображается в карточке клиента.
- [ ] PDF финмодели генерируется с корректным расчётом платежа (юнит-тесты пройдены).
- [ ] Автопостинг в Telegram-чат работает с учётом фичефлагов и настроек приватности.
- [ ] Наставник видит сводку по потоку, очередь проверок, метрики.
- [ ] PostHog принимает 11 событий, дашборды отображают данные.
- [ ] e2e-сценарии:
  1. Привязка бота → команда `/touch +1 Марат` → действие появилось в дашборде с привязкой к клиенту Марат.
  2. Сделка подтверждена → агент получает push в TG и email.
  3. Финмодель сгенерирована → PDF доступен по ссылке + расчёт совпадает с курсом.

## 6.10. Риски и решения

| Риск | Решение |
|------|---------|
| Telegram-бот падает / заблокирован у пользователя | Все события дублируются в email и в дашборд |
| Генерация PDF медленная | Очередь через Edge Function + статус «генерируется» |
| Шаблон фиксации меняется по застройщикам | В V2 — несколько шаблонов под застройщиков |
| Автопостинг раздражает | Настройки приватности у агента + фичефлаг у когорты |
| PostHog съедает бесплатный лимит | Умеренная агрегация, не каждый клик |
