# 3. Критерии «сдано» по неделям

## Общие правила приёмки

1. Каждое ДЗ имеет статус: `draft → review → accepted | rework`.
2. Наставник принимает только если **все обязательные пункты** выполнены. Желательные — на уровень оценки качества, не на блокировку.
3. На статусе `rework` фиксируется текстовый фидбэк и список конкретных правок.
4. После 2-й итерации `rework` — обязательная 15-минутная встреча с наставником.
5. ДЗ считается «принятым в срок», если `accepted_at < deadline + 24ч`.

## Неделя 0. Знакомство

**Цель:** активация в системе и фиксация целей.

### ДЗ 0.1. Подписки на 3 канала агентов

Обязательно:
- [ ] Указаны 3 ссылки на каналы/аккаунты в Instagram или Telegram.
- [ ] Для каждого — 1 предложение «что показалось интересным / что хочу повторить».

### ДЗ 0.2. Самооценка «Почему клиент должен работать со мной»

Обязательно:
- [ ] Текст на 3 предложения (не больше).
- [ ] Без штампов «индивидуальный подход», «честность» (триггер на rework — наставник проверяет).
- [ ] Виден сегмент клиента, для которого автор полезен.

### Артефакт: Карточка целей участника

Обязательно (поля заполнены, не пустые и не «—»):
- [ ] Через 8 недель я хочу уметь — конкретный навык, не «продавать».
- [ ] Главный страх / барьер.
- [ ] Сколько сделок по новостройкам в месяц — число.
- [ ] Что готов делать каждый день — конкретные действия.

## Неделя 1. Рынок, продукт, роль агента

### ДЗ 1.1. Топ-100 (10 контактов)

Обязательно:
- [ ] 10 имён в системе (мини-CRM или таблица).
- [ ] Для каждого: «почему может быть интересна новостройка» (1 строка).
- [ ] Привязан подходящий оффер из 6 шаблонов Недели 7 (или свой).

### ДЗ 1.2. Карточки 3 ЖК

Обязательно для каждой из 3 карточек:
- [ ] Название ЖК, застройщик, район, срок сдачи.
- [ ] Цены минимум по 2 типам планировок.
- [ ] Минимум 2 ипотечные программы со ставками.
- [ ] 3 плюса и 3 минуса.

### ДЗ 1.3. Доработать «Мою ценность»

Обязательно:
- [ ] Формат: «Помогаю [кому] + [что получает клиент] + [чем отличаюсь]».
- [ ] Сегмент клиента конкретен (не «всем»).
- [ ] Отличие сформулировано без сравнения с «другими агентами вообще».

### Артефакты Недели 1

- [ ] Персональная карта рынка: 5 ЖК + аргументация (по 3 аргумента на ЖК).
- [ ] Формулировка ценности агента (закреплена в профиле дашборда).

## Неделя 4. Работа с клиентом

### ДЗ 4.1. Диагностика реального клиента

Обязательно:
- [ ] Создана карточка клиента в дашборде.
- [ ] Заполнены ответы по чек-листу из 10 вопросов курса.
- [ ] Заполнены поля `Push / Pull / Anxiety / Habit` (минимум по 1 примеру в каждое).
- [ ] Указан предложенный «следующий шаг» (встреча/подборка/пауза).

### ДЗ 4.2. Фиксация уникальности

Обязательно:
- [ ] Шаблон фиксации заполнен по всем полям курса.
- [ ] Привязан к карточке клиента в дашборде.
- [ ] Если уникальность реальная — статус «отправлена» или «подтверждена».
- [ ] Если учебная — пометка «учебный пример».

### ДЗ 4.3. Доработка карточек ЖК

Обязательно:
- [ ] Минимум 1 карточка ЖК обновлена за последнюю неделю (поле `updated_at`).

### Артефакты Недели 4

- [ ] Персональный чек-лист диагностики (10+ вопросов).
- [ ] 2 заполненных шаблона фиксации уникальности.

## Неделя 5. Ипотека и страховка

### ДЗ 5.1. Изучение 3 банков-партнёров

Обязательно:
- [ ] Таблица с 3 банками: ставки по семейной и стандартной, ПВ, лимит.
- [ ] Указан источник данных (ссылка / дата выгрузки).

### ДЗ 5.2. Финмодель для клиента

Обязательно:
- [ ] Заполнен шаблон финмодели курса (2–3 варианта в одной таблице).
- [ ] Привязка к реальному или учебному клиенту.
- [ ] Указаны: ЖК, тип квартиры, стоимость, ПВ, кредит, программа, ставка, срок, платёж/мес., плюсы, минусы.
- [ ] Платёж рассчитан корректно (наставник проверяет на ошибки в формуле).

### ДЗ 5.3. Стоимость страхования

Обязательно:
- [ ] Стоимость комплексного страхования для кредита 4 млн от 2 страховых компаний.
- [ ] Указана годовая стоимость и ежемесячный эквивалент.

### ДЗ 5.4. Шпаргалка по ипотечным программам

Обязательно:
- [ ] 1 страница (PDF или MD).
- [ ] Минимум 4 программы: семейная, стандартная, IT, без ПВ.
- [ ] Для каждой: ставка, ПВ, кому подходит, ключевые условия.

### Артефакты Недели 5

- [ ] 4 финансовых карточки клиентов (по учебным кейсам).
- [ ] 1 полная финмодель (2–3 варианта).

## Неделя 6. Перерыв и самостоятельная отработка

**Особенность:** не проверяется наставником лично, но автоматический чек-лист в дашборде.

### Чек-лист недели 6 (дашборд проверяет автоматически)

- [ ] Все карточки ЖК заполнены и актуализированы — минимум 11 (5 Уфа + 6 регионы).
- [ ] Шпаргалка по ипотечным программам в `accepted`.
- [ ] Финмодель для 1 клиента в `accepted`.
- [ ] Из Топ-100 написано минимум 10 контактам — счётчик `action: dialog`.
- [ ] Проведена минимум 1 диагностика реального клиента (карточка клиента + 10 ответов).
- [ ] Зафиксирована минимум 1 уникальность (учебная или реальная).
- [ ] Посещён 1 вебинар застройщика (ссылка + конспект).

XP за неделю 6 начисляется автоматически только при выполнении всех 7 пунктов.

## Неделя 8. Стратсессия №2

### ДЗ 8.1. Полный цикл с реальным клиентом

Обязательно:
- [ ] В дашборде есть клиент с заполненной диагностикой.
- [ ] К нему привязана финмодель.
- [ ] Зафиксировано минимум 3 касания.
- [ ] Указан результат: «следующий шаг» или «отказ + причина».

### ДЗ 8.2. Ответы на 5 возражений по ипотеке

Обязательно:
- [ ] 5 возражений из курса (рабство 25 лет, подожду снижения ставок, переплата, нет ПВ, потеря работы).
- [ ] К каждому — ответ агента своими словами (не копия из курса).
- [ ] В каждом ответе — минимум 1 цифра / факт.

### ДЗ 8.3. Актуализация ипотечных условий

Обязательно:
- [ ] В минимум 3 карточках ЖК обновлены ставки/программы (поле `updated_at` < 7 дней).

### Артефакты Недели 8

- [ ] Результат комплексной симуляции (оценка наставника по 6 критериям).

## Неделя 9. Система работы и план на 150к

### ДЗ 9.1. Запустить недельный план

Обязательно:
- [ ] В дашборде заполнен план на следующую неделю (по дням).
- [ ] За первую неделю выполнения — факт по всем 5 типам действий не менее 70% от плана.

### ДЗ 9.2. Дашборд по итогам первой недели

Обязательно:
- [ ] Скрин или экспорт «факт vs план» по INPUT-метрикам.
- [ ] Заполнены 6 строк таблицы каналов.
- [ ] Краткий вывод: какой канал дал диалоги, где провал.

### ДЗ 9.3. Поддержка каналов

Обязательно:
- [ ] Минимум 10 новых касаний за неделю (счётчик `action: touch`).
- [ ] Минимум 3 новых сторис.

### ДЗ 9.4. Обновить базу знаний

Обязательно:
- [ ] Минимум 1 новая карточка ЖК добавлена в общую базу команды (статус `accepted`).

### Артефакты Недели 9 (финальные)

- [ ] Персональный план на 150к (расчёт воронки) — с цифрами по конверсиям.
- [ ] Недельное расписание — заполнены все 7 дней.
- [ ] Дашборд метрик на первый месяц.
- [ ] 3 публичных обязательства (зафиксированы в профиле и в чате потока).
