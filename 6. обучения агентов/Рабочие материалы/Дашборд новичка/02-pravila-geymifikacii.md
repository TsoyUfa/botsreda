# 2. Правила геймификации

## 2.1. Принципы

1. XP начисляется только за реальные действия — не за «открытие вкладки».
2. Действия с дневным капом — нельзя «накрутить» 100 касаний за час.
3. Сделки и монетки начисляются только после подтверждения наставника или РОПа.
4. Один источник правды: значения хранятся в `xp_events`, ранг пересчитывается агрегатом.
5. Ранги — постоянные, не сгорают. Монетки — могут сгорать в бонус (одна монетка тратится только на один бонус).

## 2.2. XP за обучение

### Сдача ДЗ недели

| Неделя | XP за сдачу ДЗ | Обоснование |
|--------|----------------|-------------|
| 0 | 100 | минимальная активация |
| 1 | 200 | карта рынка + ценность агента |
| 2 | 250 | 6 карточек по регионам — объёмнее |
| 3 | 150 | стратсессия — закрепление |
| 4 | 300 | диагностика + фиксация уникальности — ключевой навык |
| 5 | 300 | ипотека и финмодель — ключевой навык |
| 6 | 100 | перерыв, чек-лист самостоятельной работы |
| 7 | 250 | каналы привлечения + рассылки |
| 8 | 200 | стратсессия №2 |
| 9 | 300 | план на 150к + дашборд метрик |
| **Итого** | **2 150** | сумма за пройденный курс без артефактов |

### XP за принятый артефакт

| Тип артефакта | XP | Комментарий |
|--------------|----|-------------|
| Карточка ЖК | 30 | повторяемый артефакт, поэтому небольшой XP |
| Финмодель | 100 | сложный артефакт |
| Шаблон фиксации уникальности | 50 | привязан к реальному клиенту |
| Скрипт / рассылка | 40 | проверяется наставником на качество |
| Диагностика клиента (10 вопросов) | 80 | требует реальной работы с клиентом |
| Шпаргалка по ипотеке | 60 | повторно используется |
| Сценарий 3 сторис | 30 | креативный артефакт |
| План на 150к (личная воронка) | 150 | финальный артефакт программы |

> Артефакты сверх обязательных (например, новичок добровольно сделал 10 карточек ЖК вместо 5) идут с тем же XP, но с месячным капом 1 000 XP, чтобы не было фарма.

## 2.4. XP за сделки

| Событие | XP |
|---------|-----|
| Бронь / задаток | 200 |
| Подписан договор | 300 |
| Деньги получены | 500 |
| **Полный цикл одной сделки** | **1 000** |

Сделка считается «оплаченной» после подтверждения РОПом — только тогда XP за этап «деньги получены» начисляется.

## 2.5. Ранги

| Ранг | Порог XP | Что значит |
|------|----------|------------|
| Стажёр | 0 | старт программы |
| Агент | 1 000 | прошёл базу (недели 1–2 + первые действия) |
| Опытный | 3 000 | прошёл первые 5 недель + 1 сделка или сильная активность |
| Мастер | 7 000 | завершил программу + 2–3 сделки |
| Легенда | 15 000 | системно работающий агент потока |

### Расчёт

```ts
function calculateRank(totalXp: number): RankCode {
  if (totalXp >= 15_000) return 'legend'
  if (totalXp >= 7_000)  return 'master'
  if (totalXp >= 3_000)  return 'experienced'
  if (totalXp >= 1_000)  return 'agent'
  return 'trainee'
}
```

### Балансировка по сценарию

Проверка: новичок, идущий по программе без сделок, накапливает за 9 недель:
- ДЗ: 2 150
- Артефакты (минимум): ~600
- Действия (4 недели активной работы по плану): ~1 600
- Streak: ~300
- **Итого без сделок: ~4 650 XP → ранг «Опытный»** ✅

С 1 сделкой добавляется 1 000 XP → 5 650 (всё ещё «Опытный»).
С 2 сделками → 6 650 (близко к Мастеру).
С 3 сделками → 7 650 → **«Мастер»** ✅

Гипотеза подтверждается: пройти курс и сделать 2–3 сделки = выйти на «Мастера». Корректируем по факту первого потока.

## 2.6. Монетки и бонусы

### Что такое монетка

**1 монетка = 1 закрытая и оплаченная сделка**, подтверждённая РОПом. Источник идеи — [Идеи на реализацию.md, раздел 4.2](Идеи%20на%20реализацию.md).

### Шкала бонусов (фиксируем!)

| Порог монеток | Бонус | Сумма / условие | Тип |
|---------------|-------|-----------------|-----|
| 3 монетки | Бонус «Стартер» | +5 000 ₽ к первой выплате после 3-й сделки | разовый |
| 5 монеток | Бонус «Системный» | +15 000 ₽ к выплате после 5-й сделки | разовый |
| 10 монеток | **Перевод на 65/35** | новые условия комиссии (65% агенту) | постоянный |
| 20 монеток | Бонус «Амбассадор» | +50 000 ₽ + статус амбассадора | разовый |

> Суммы бонусов — стартовая гипотеза. Перед запуском пилота — согласовать с финансовой моделью агентства.

### Логика «расхода» монеток

- Монетки **не списываются** при получении бонусов «Стартер» и «Системный» — это бонус «за достижение», а не «обмен».
- Переход на 65/35 — постоянный, монетки тоже не сбрасываются.
- В дашборде монетки видны как «жетоны достижений», а не валюта.

### Спец-монетки (опционально, обсуждаемо)

| Источник | Условие | Кому достаётся |
|----------|---------|----------------|
| Монетка «Контент» | Reels/пост набрал 100+ реакций | автору |
| Монетка «Наставник» | Помог коллеге закрыть сделку (подтверждено) | помощнику |
| Монетка «Челлендж» | Победа в недельном челлендже потока | победителю |

> Спец-монетки идут в общий счётчик, но визуально подсвечиваются другим цветом. Решение по их включению — после первого потока.

- Заблокированные ачивки видны: «что нужно сделать, чтобы получить» — стимулирует.
- Каждая ачивка — это одновременно XP-событие в `xp_events` (источник `achievement`).
- Ачивки нельзя получить дважды.

## 2.8. Антифрод

### Жёсткие правила (на уровне БД и Edge Functions)

1. **Дневные капы XP** — задаются в `config/xp.ts`, проверяются перед записью в `xp_events`.
2. **Минимум полей для действия:**
   - касание / диалог / подборка / показ — обязателен `client_id` или `channel + comment` (не пусто, не «—», не повтор предыдущего).
   - контент — обязательна ссылка или скриншот.
3. **Анти-серия:** если 5 действий одного типа создаются в течение 1 минуты — последние 4 идут со статусом `pending_review`.
4. **Сделки:** XP за «деньги получены» и монетка начисляются только Edge Function-ом `confirm_deal`, который запускается при `status = 'paid'` и наличии `confirmed_by` с ролью `mentor`/`admin`.
5. **Артефакты:** XP начисляется только при `status = 'accepted'`. Любые правки → возврат в `draft`, XP не снимается, но не начисляется повторно.

### Мягкие правила (на уровне UI и наставника)

- Лента активности у наставника подсвечивает аномалии: «> 30 касаний за час», «10 диалогов без комментариев».
- Наставник может отозвать XP вручную с фиксацией причины — событие в `xp_events` с отрицательной суммой и `source_type = 'manual'`.
