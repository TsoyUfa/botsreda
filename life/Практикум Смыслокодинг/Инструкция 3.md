🚀 Этап 1. GitHub - код в безопасности · ~10 мин▸

GitHub - облако для кода, помнит каждое изменение. Сломалось - «верни как было».

Четыре слова, которые нужно знать:

- **Репозиторий** - папка проекта в облаке
- **Коммит** - сохранение (как Ctrl+S, только для всего проекта)
- **Пуш** - отправка сохранений в облако
- **Приватный / открытый** - приватный репозиторий видишь только ты, открытый видит весь интернет (любой может скопировать). Для своего проекта на старте - **всегда приватный**: в коде могут оказаться промежуточные секреты, недоделки, ключи. Открытый - это когда ты осознанно делишься своим решением с миром (open source)

### Шаг 1.1. Зарегистрируйся на GitHub

- Зайди на [github.com](https://github.com/) и создай аккаунт (если ещё нет)
- Сохрани логин и пароль

### Шаг 1.2. Подключи проект к GitHub

Открой Claude Code в папке своего проекта. Скопируй и вставь:

ПРОМПТ ДЛЯ АГЕНТА📋 Копировать промпт

Подключи этот проект к GitHub. 1. Если у меня ещё нет репозитория - создай ПРИВАТНЫЙ репозиторий на GitHub. Назови его по имени моего проекта. 2. Перед коммитом убедись, что в .gitignore есть .env и любые другие файлы с секретами (ключи, токены, пароли). Если их нет - добавь. 3. Сделай первый коммит со всем кодом проекта, КРОМЕ .env и файлов из .gitignore. 4. Запуши в GitHub. 5. Покажи мне ссылку на репозиторий. Если нужна авторизация GitHub - объясни пошагово, что мне нажать.

**Проверка:** открой ссылку на репозиторий в браузере. Видишь свои файлы? `.env` НЕ видишь? Значит код в безопасности, ключи не утекли.

> ⚠️ Один раз закоммитил `.env` в публичный репо - и всё, ключи в истории навсегда (даже если потом удалить файл - в коммитах он останется). Чистка истории долгая и нервная. Проще не пускать туда `.env` с самого первого коммита.

### Шаг 1.3. Попробуй откат (по желанию)

Хочешь убедиться, что откат работает? Скажи Claude:

ПРОМПТ ДЛЯ АГЕНТА📋 Копировать промпт

Удали главную страницу проекта.

Он удалит. А потом - **любая из разговорных формул**, без технического жаргона:

- «Верни как было вчера»
- «Ты что-то натворил, вчера работало - откати»
- «Откати последние изменения»

Восстановил? Откат работает. Теперь ты в безопасности.

### Шаг 1.4. Зафиксируй правило коммита в CLAUDE.md

Чтобы не вспоминать каждый раз «надо коммитить», пропиши правило в `CLAUDE.md` - Claude будет следовать ему сам. Два варианта на выбор:

**Вариант А (под контролем).** «Никогда не делай коммит, пока я не попрошу. Никогда не коммить чужие изменения.» Подходит, когда ты любишь сам ставить точки.

**Вариант Б (автомат).** «Всегда, когда заканчиваешь работу над фазой - делай коммит автоматически. Название коммита запиши в файл плана рядом с галочкой фазы.» Подходит, когда ты часто забываешь коммитить.

Готовый промт:

ПРОМПТ ДЛЯ АГЕНТА📋 Копировать промпт

Добавь в CLAUDE.md правило коммита: [вариант А или Б из инструкции]. И второе правило - после каждой фазы плана коммит обязательный, имя коммита записывай в файл плана рядом с галочкой фазы. Чтобы потом я мог в плане видеть, где какой коммит, и в случае чего откатиться к нужной точке.

### Шаг 1.5. Три правила безопасности

Это базовая гигиена. Запомни их один раз - они применяются всю жизнь:

1. **Bypass permissions (автономия Claude) - только в одной конкретной папке проекта.** Когда даёшь Claude автономию `--dangerously-skip-permissions` - это значит «делай что хочешь без моего подтверждения». Это нормально внутри одной папки твоего проекта. **Никогда** не разрешай это системно (на весь компьютер) - случайно сотрёт что-то важное.
2. **Один раз в день коммит - обязательно. Лучше после каждой фазы.** Каждый вечер. Минимум. Утром у тебя должна быть точка отката.
3. **Секреты лежат в `.env`, и `.env` прописан в `.gitignore`.** Никаких ключей в коде. Никогда не вставляй ключи в публичный чат, скриншот или стрим. Даже временные. Утечка ключа = чужая трата с твоего счёта.

🌍 Этап 2. Деплой - сайт в интернете · ~15 мин▸

**Четыре пути:**

- **Vercel** - лендинг или простой сайт без базы и бэкенда. Самый быстрый, бесплатно.
- **Railway** - международный рынок, есть бэкенд, база, фоновые задачи. От $5/мес.
- **Amvera Cloud** - для российских пользователей (152-ФЗ, хранение данных в РФ).
- **TimeWeb** - альтернатива Amvera. Российский хостинг, оплата картой РФ, есть как российские, так и зарубежные серверы в одном кабинете.

Как выбрать:

- Нет базы данных, нет оплат, не для россиян - **Vercel**
- Международный проект с бэком - **Railway**
- Российские пользователи, персональные данные - **Amvera** или **TimeWeb**
- Не знаешь - бери **Amvera**, потом при необходимости перенесёшь.

> 🌍 У Amvera и TimeWeb можно брать **зарубежные серверы** в российском кабинете с оплатой картой РФ - удобно для международной аудитории без зарубежной карты.

> ⚠️ **ENV-переменные при деплое.** Локально `.env` подгружается автоматически, на проде - **нет**. После деплоя перенеси ключи (Claude, OpenRouter, БД, платёжки) в настройки хостинга. Скажи Claude Code: «перенеси все ENV-переменные из локального .env в настройки [Vercel / Railway / Amvera]» - он сделает через CLI или напишет промт для Chrome Claude (см. Этап 3). Без этого публичная версия упадёт на первом же обращении к Claude API.

Путь А. Vercel (самый быстрый)▸

> 🚨 **Vercel - иностранный сервис. Если собираешь email/телефон/имя россиян - Vercel НЕ подходит.** По 152-ФЗ персональные данные россиян хранятся только на российских серверах. Для регистрации россиян бери Amvera или TimeWeb.

🛡 Этап 2.5. Прогон на дыры перед публикацией · ~10 мин▸

Прежде чем массово показывать сайт людям - попроси Claude найти всё, что может пойти не так.

Открой Claude Code и вставь:

ПРОМПТ ДЛЯ АГЕНТА📋 Копировать промпт

Найди 10 причин, по которым этот проект может упасть, опозорить меня или подвести пользователей. Думай как пессимист, который ищет проблемы: 1. Безопасность: где ключи API могут оказаться на клиенте? Есть ли дыры в авторизации? Доступ к админке защищён? 2. Нагрузка: что будет, если зайдёт 1000 человек одновременно? Где упадёт? 3. Юридика: что собираем у пользователей? Где-то надо уведомление Роскомнадзора? Согласие на ПДн? Оферта? 4. Функционал: какие кнопки не работают, какие формы не валидируются, какие сценарии я не закрыл? 5. Мобильная вёрстка: как выглядит на телефоне? Что ломается? 6. SEO/индексация: тайтлы, мета-описания, robots.txt - всё на месте? 7. Бэкапы: что произойдёт, если БД упадёт прямо сейчас? 8. Стоимость: где сайт может неожиданно жечь деньги (LLM-запросы, трафик, фоновые задачи)? 9. UX: где пользователь запутается на 3 секунде и закроет вкладку? 10. Что-то, чего я сам не вижу - подумай шире. Для каждой причины - объясни, в чём риск, и сразу предложи фикс. Будь жёстким, не утешай меня.

> Часть дыр пофикси сразу, часть выпиши в `plans/` как отдельные задачи. Не пытайся закрыть всё за один день.

🌐 Этап 3. Chrome Claude - браузерный помощник · ~10 мин▸

Связка Claude Code + Chrome Claude: Claude Code пишет код, Chrome Claude (расширение в браузере) сам кликает, заполняет формы, регистрируется в сервисах, настраивает личные кабинеты.

### Шаг 3.1. Установи расширение

1. Открой Google Chrome (если ещё нет - скачай [google.com/chrome](https://www.google.com/chrome/))
2. Зайди в Chrome Web Store, найди «Claude» (официальное расширение от Anthropic)
3. Нажми «Установить», авторизуйся в свой аккаунт Claude
4. В настройках расширения выбери модель: **Sonnet** для простых задач, **Opus** для сложных
5. Режим: поставь **«Ask before acting»** - Claude будет спрашивать перед каждым действием (на первое время безопаснее)

> Не знаешь, как установить? Спроси у Claude Code: ```prompt Мне нужно расширение Claude для Google Chrome. Дай прямую ссылку на установку и объясни пошагово, как настроить (авторизация, выбор модели, режим работы).` ``

### Шаг 3.2. Боевой кейс - настрой Яндекс.Метрику

Задача: поставить счётчик Яндекс.Метрики на свой задеплоенный сайт, не заходя самому в интерфейс Метрики.

> ⚠️ Нужен задеплоенный сайт из Этапа 2. Если деплой не получился - пропусти этот шаг, вернёшься после того, как сайт будет в интернете.

**Шаг первый.** Открой Claude Code, скажи волшебную фразу:

ПРОМПТ ДЛЯ АГЕНТА📋 Копировать промпт

Мне нужно поставить счётчик Яндекс.Метрики на мой сайт. Напиши мне промт для Chrome Claude, который: 1. Зайдёт на metrika.yandex.ru 2. Создаст новый счётчик с адресом моего сайта [вставь сюда ссылку на свой задеплоенный проект] 3. Настроит цели (базовые: просмотр страницы, клик по кнопке) 4. Скопирует код счётчика Промт должен быть очень подробный, пошаговый, чтобы Chrome Claude точно всё сделал. После этого скажи, что я должен сделать сам в Claude Code, чтобы вставить этот код на сайт.

**Шаг второй.** Claude Code напишет тебе промт для Chrome Claude. Скопируй его.

**Шаг третий.** Открой metrika.yandex.ru в Chrome, нажми на иконку Claude-расширения, вставь промт и смотри, как он сам кликает, регистрирует, создаёт счётчик.

**Шаг четвёртый.** Скопируй код счётчика (Claude его выдаст), возвращайся в Claude Code:

ПРОМПТ ДЛЯ АГЕНТА📋 Копировать промпт

Вот код счётчика Яндекс.Метрики: [вставь код]. Встрой его в мой сайт правильно - в head или в layout, чтобы он загружался на всех страницах. Закоммить и запушь.

**Проверка:** зайди в metrika.yandex.ru, открой свой счётчик. Через 5-10 минут после пуша там должны появиться данные о посещениях (хотя бы о твоём собственном заходе на сайт).

### 🪄 Универсальная формула

**Когда Claude Code просит у тебя что-то, что лежит в чужом веб-интерфейсе** (хостинг, аналитика, платёжка, рекламный кабинет, банковский ЛК, CRM) - **не открывай его сам**. Скажи:

ПРОМПТ ДЛЯ АГЕНТА📋 Копировать промпт

Не знаю, где это взять. Напиши мне промт для Chrome Claude, который зайдёт в [название сервиса], найдёт [что нужно: DATABASE_URL / API-ключ / webhook URL / счёт] и скопирует мне обратно.

Как устроен любой проект▸

Три слоя:

|Слой|Что это|Где живёт|
|---|---|---|
|**Зачем**|Бизнес-контекст: кто клиент, что продаём, цели|`.business/`|
|**Как**|Технические планы|`plans/`|
|**Что**|Сам код|`src/`, `prisma/` и т.д.|

`CLAUDE.md` - навигация, Claude читает её первой. **`ai-clone/`** - надпроектная папка (одна на всю жизнь): принципы, цели. **`triggers/`** - автоматизации, когда проект подрастает.

Для большого проекта (SaaS, стартап) добавляются `idea/` (research) и `architecture/` (чертёж: стек, БД, фронт/бэк). Для лендинга / MVP - хватит `CLAUDE.md` + `plans/` + код.

Цикл работы▸

1. **Утро.** «Прочитай CLAUDE.md и .business/goals/monthly.md. Что у нас в фокусе?»
2. **Задача.** Формулируй конкретно: не «сделай сайт», а «Добавь форму регистрации на лендинг. Email + имя. После отправки - редирект на /thankyou».
3. **План.** «Напиши план реализации в plans/». Затем: «У нас в idea.md лежит проработанная идея. Проверь, что упустил в плане, и допиши».
4. **Работа.** Claude пишет код, ты проверяешь в браузере. Если не нравится - скриншот + голос.
5. **Коммит.** «Коммит и пуш. Имя коммита запиши в файл плана рядом с галочкой фазы».
6. **Обновление.** В конце сессии: «Обнови план, отметь что сделано».

Как Claude пишет план: 4 прохода▸

1. **Первичный план.** «Напиши план реализации фичи X. Фазы, чекбоксы».
2. **Дополненный.** «Прочитай свой план. Что упустил? Допиши».
3. **Ещё раз дополненный.** «Чего не хватает с точки зрения архитектуры / интеграций / тестов? Допиши».
4. **Выпил лишнего.** «Жёстко выпили всё, что для MVP не нужно».

Как реализовать план: оркестратор + исполнитель▸

Контекст Claude забивается за пару файлов - после 40-60 тыс. токенов он теряет точность. Поэтому план разбивается на фазы, каждая реализуется в новом чате.

**Схема двух чатов:**

- **Чат-оркестратор** держит план целиком. Читает план, даёт тебе промт следующей фазы, проверяет результат предыдущей.
- **Чат-исполнитель** - чистый новый чат. В нём реализуешь ОДНУ фазу. Готово - закрыл. Следующая фаза - новый чистый чат.

Промт оркестратору в начале:

ПРОМПТ ДЛЯ АГЕНТА📋 Копировать промпт

На каждую фазу написать мне промт реализации - короткий, с минимальным контекстом, чтобы я мог запустить его в пустом новом чате. #### ❌ Чего не делать: мульти-агентные оркестраторы

Не запускай 100 агентов параллельно (claude-flow и подобные). Агенты переписывают друг друга, на выходе - каша, которую невозможно починить. Управляй одной фазой за раз сам как архитектор.

#### Две стратегии реализации

**Режим А - под контролем.** Каждая фаза отдельно, ты проверяешь результат. Для проекта с оплатой / БД / клиентскими данными.

**Режим Б - автономно.** «Разберись сам, выполни все фазы, позови меня, когда закончишь». Для маленькой фичи или одностраничника.

🔁 Микро-ритуал между фазами: проверка + security review▸

Готовый промт-шаблон:

ПРОМПТ ДЛЯ АГЕНТА📋 Копировать промпт

Фаза [N] закончилась. 1. Проверь на ошибки то, что ты сам сделал в этой фазе. Не лезь в чужие файлы - только то, что трогал. 2. Запусти security review на код этой фазы. 3. Если нашлись проблемы - покажи список и сразу исправь. 4. Когда всё чисто - дай мне промт следующей фазы.

**Security review - официальный скилл от Anthropic.** Один раз поставь его в проект:

ПРОМПТ ДЛЯ АГЕНТА📋 Копировать промпт

Найди и установи официальный skill `security-review` от Anthropic для Claude Code. После установки - запусти на текущем коде проекта и покажи, что нашлось.

**От чего защищает security review:**

- ключи API в коде фронт-энда (видны через DevTools)
- дыры в авторизации
- публичная документация API наружу
- открытые админки без проверки прав

🎼 Финальная проверка перед запуском▸

ПРОМПТ ДЛЯ АГЕНТА📋 Копировать промпт

Все фазы плана закрыты. Прогон финальной проверки перед запуском: 1. Запусти security review на всём проекте, не только на последней фазе. 2. Прогон «10 причин обосраться» (см. промт в Этапе 2.5) на полном функционале. 3. Проверь: все ENV-переменные на проде проставлены, бэкапы БД настроены, оплата (если есть) идёт через webhook, ключи нигде не утекли в код фронта. 4. Покажи мне финальный чек-лист «всё проверено, готово к запуску» / «есть вот эти проблемы, надо устранить».

Особенно важно, если в проекте есть **оплата, ПДн или авторизация**.

Главные правила▸

- **Одна задача за раз.** «Сделай лендинг». Потом «добавь оплату». Потом «подключи рассылку».
- **Скриншот вместо 1000 слов.** Не нравится кнопка - скриншот, обведи, скажи что не так.
- **Не бойся сломать.** У тебя есть GitHub. Сломал - откатил.
- **Claude не телепат.** «Тёмный фон, белый текст, кнопка со свечением» лучше, чем «сделай красиво».
- **Чем сложнее проект - тем больше контроль.** SaaS с оплатой и БД: после каждой фазы прогон на критика, после каждой фичи - проверка дыр.
- **Спрашивай у Claude, а не у наставника.** По техническому выбору (база, стек, реализация) первый вопрос - Claude.
- **Финальная прокатка глазами обязательна.** «Исходя из этого всего, человеческим языком одной строкой: ключевая идея, в двух строках: зачем оно нам с учётом моего проекта».
- **Ты архитектор, не исполнитель.** Не лезь в код руками. Твоя работа - архитектура: что за чем, какие слои, где БД, где API, где фронт.

💬 Как формулировать промты▸

**Чем точнее ты представляешь результат - тем лучше промт.**

### Четыре элемента хорошего промта

**1. Контекст** - кто ты и что за проект.

> «Я делаю сайт для доставки еды в Казани»

**2. Задача** - что конкретно нужно сделать.

> «Добавь страницу с меню»

**3. Детали** - как именно должно выглядеть и работать.

> «Карточки блюд с фото, ценой и кнопкой "В корзину". Фильтр по категориям: пицца, суши, бургеры»

**4. Ограничения** - чего НЕ делать.

> «Без авторизации пока. Корзину не делай, только кнопки»

### Примеры: плохо vs хорошо

**Плохо:** «Сделай мне сайт» **Хорошо:**

ПРОМПТ ДЛЯ АГЕНТА📋 Копировать промпт

Создай лендинг для моего проекта. Одна страница. Блоки сверху вниз: 1. Hero - заголовок, подзаголовок, кнопка CTA. 2. Проблема - 3 боли целевой аудитории. 3. Решение - что мы предлагаем и почему это работает. 4. Как это работает - 3 шага. 5. Отзывы - 3 карточки. 6. CTA - форма с email и кнопкой. Возьми тексты из .business/products/ и .business/audience/. Дизайн: тёмный фон, минималистичный, современные шрифты.

**Плохо:** «Исправь баг» **Хорошо:**

ПРОМПТ ДЛЯ АГЕНТА📋 Копировать промпт

При нажатии на кнопку «Отправить» ничего не происходит. Вот скриншот. В консоли браузера ошибка: «TypeError: Cannot read property 'email' of undefined». Найди причину и исправь.

**Плохо:** «Сделай красиво» **Хорошо:**

ПРОМПТ ДЛЯ АГЕНТА📋 Копировать промпт

Мне не нравится, как выглядит эта секция. Вот скриншот. Проблемы: 1. Слишком много текста, нечитаемо. 2. Кнопка теряется на фоне. 3. Отступы маленькие, всё слипается. Переделай: сократи текст до 2 предложений, кнопку сделай контрастной со свечением, добавь отступы между блоками.

### Приёмы

**«Сначала план».** «Не пиши код. Сначала опиши план. Я проверю и дам добро».

**«Покажи варианты».** «Предложи 3 варианта, как это можно сделать. Кратко, по 2 предложения каждый».

**«Одно за раз».** Не проси в одном промте и дизайн, и базу, и оплату. Одна задача - один промт.

**«Жёсткая критика под MVP».** «Жёстко выпили из плана всё, что для MVP не нужно».

**«Оспаривай оценки Claude».** Когда слышишь от него «6 недель и 300 тысяч» - оспаривай: «ты уверен? как считал? убери из расчёта стоимость операторов - я один».

**«Как было / как станет».** В начале фазы проси: «сформулируй как было до этой фазы и как станет после».

**«Очередь задач».** «Когда закончишь это - сразу стартуй вот это».

**«Срежь дистанцию».** «Дожми остальные фазы сам, дай мне ссылку на результат».

**«AI как критик».** «Какие из этих идей реально хорошие, а какие шлак, и почему. Доказывай по каждой».

❓ FAQ▸

### У меня ничего не деплоится, ошибки

Скопируй ошибку и отправь Claude. Скажи: «Вот ошибка при деплое. Разберись, почему, и исправь». В 90% случаев он сам починит.

### Я не понимаю, что такое PostgreSQL / деплой / webhook

И не надо. Говоришь Claude «подключи базу» - он подключает. Твоя задача - формулировать намерение.

### Vercel, Railway или Amvera - что выбрать?

- Лендинг / простой сайт без базы - **Vercel** (бесплатно, 2 минуты)
- Международный проект с бэком - **Railway**
- Российские пользователи, персональные данные - **Amvera** (152-ФЗ)
- Не знаешь - бери **Amvera**, потом при необходимости перенесёшь

### Бесплатные варианты для деплоя есть?

- **Vercel** - для простых сайтов без бэкенда, полностью бесплатный
- **Amvera** - 111 руб. при регистрации, хватит попробовать
- **Railway** - есть бесплатный уровень с ограничениями

### Chrome Claude опасен?

Поставь режим **«Ask before acting»** - спрашивает перед каждым действием. Не давай ему делать необратимые вещи (удаление аккаунтов, подтверждение платежей) без подтверждения.

### Как получить красивый дизайн, если первый промт не помог?

**Скриншот** (Этап 5.1): сайт-референс → «укради этот дизайн». Если не помогло - **конкретный компонент** (Этап 5.2): зайди на [ui.aceternity.com](https://ui.aceternity.com/), назови имя Claude (Spotlight / Aurora / Meteors).

### Я из России, Claude блокирует

Французская e-SIM (см. День 1). **Amvera** проксирует запросы к Claude API с российских серверов. Альтернатива - **DeepSeek** (в 5 раз дешевле Sonnet, ~90% качества Opus).

### Claude завис или тормозит

1. Скопируй последний промт.
2. Открой новый чат в Claude Code, вставь промт заново.
3. Держи ChatGPT-5 как fallback для критических кусков.

### Claude или ChatGPT для планирования - что лучше?

Для длинных планов - **Claude**. Контекстное окно больше, не выдыхается в середине.

### Мульти-агентные системы / claude-flow - будут?

Нет. Один проект, одна фаза за раз. Разбор - в «❌ Чего не делать» выше.
