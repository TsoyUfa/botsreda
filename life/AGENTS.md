# 🤖 ИНСТРУКЦИЯ ДЛЯ ИИ-АГЕНТОВ (AGENTS.md)

> **Статус:** Главный конфигурационный файл базы знаний  
> **Цель:** Дисциплинировать любого ИИ-агента, подключающегося к этому репозиторию (Cursor, Claude Code, боты), и превратить его из чат-бота в системного администратора Второго Мозга.

---

## 📂 1. Системная структура папок

Любые новые документы ИИ должен строго распределять по **4 прикладным папкам** (Квадрантам РОСТ):

1. **`1. Бизнес`** — Всё, что приносит деньги. Описание продуктов, тарифы, портреты клиентов, коммерческие предложения.
   * *Главный вопрос:* «Что мы продаем и по какой цене?»
2. **`2. План`** — Задачи и планирование. Входящие задачи, календари, дорожные карты проектов, график публикаций.
   * *Главный вопрос:* «Что конкретно нужно сделать сегодня и на этой неделе?»
3. **`3. Мой клон`** — Личность собственника и стандарты системы. Манера речи (ToV), характер, инструкции для ИИ-помощников, файлы стиля и скрипты автоматизации.
   * *Главный вопрос:* «Каковы ключевые ценности и принципы общения собственника?»
4. **`4. Мастерство`** — Стандарты качества. Инструкции по личной эффективности, чек-листы для сотрудников, готовые ИИ-запросы, шаблоны регламентов.
   * *Главный вопрос:* «Как сделать бизнес-процесс быстро и на высоком уровне?»

> **Правило Черновиков (`raw/`):** Все быстрые заметки, диктовки с телефона, записи со встреч или незаконченные мысли уходят в специальный раздел для черновиков `raw/` (сырье) внутри нужного раздела, чтобы не путать ИИ устаревшей информацией.

## ✍️ 2. Манера речи собственника (ToV)

Любой текст, создаваемый ИИ для внешнего мира (посты, сценарии, сообщения клиентам), должен строго соответствовать **Правилу 3П**:
1. **Не врать:** Писать только то, что реально проверено на практике. Никакой пустой теории и «воды».
2. **Не приукрашивать:** Считать результаты в рублях, часах и реальных сделках, а не в абстрактных «успехах». Связывать результаты с выходом собственника на **500к+ ₽/мес** и личную свободу.
3. **Не подлизываться:** Писать просто и честно, без корпоративного пафоса и рекламных штампов. Говорить прямо и уважительно.

**КРИТИЧЕСКОЕ ТРЕБОВАНИЕ:** Строгое отсутствие англицизмов во всех русскоязычных текстах. Заменять:
* `Inbox` ➔ `Входящие задачи`
* `Tone of Voice` ➔ `Манера речи (Голос автора)`
* `SoloOps` ➔ `Автономная экосистема`
* `Link engineering` ➔ `Смысловое связывание`
* `B2B` ➔ `Бизнес-процессы`
* `Telegram / Obsidian` ➔ `Телеграм / База знаний`

**ПУНКТУАЦИОННОЕ ТРЕБОВАНИЕ:** Полный запрет на длинные тире («—»). Во всех генерируемых текстах, регламентах и файлах использовать только короткие стандартные дефисы/тире («-»).

## 📐 3. Визуальный код оформления (РОСТ-Стиль)

Любой генерируемый PDF-документ или презентация должны соответствовать премиальному темному брутализму по стандартам файла `3. Мой клон/style/дизайн-код-РОСТ.md`:
* **Фон:** Глубокий темно-синий (`#0b1220`) для всех страниц.
* **Рамка:** Каждая страница обрамлена тонкой рамкой (`#1e293b`) со смещением 30pt от краев (эффект A4-постера).
* **Акценты:** Золотой/латунный цвет `#d8b46a` для заголовков, рамки карточек и номеров шагов.
* **Текст:** Мягкий белый `#e2e8f0` для комфортного чтения на темном фоне.
* **Обложка:** Строго без верхних и нижних колонтитулов и номеров страниц.
* **Карточки (Highlight Blocks):** Вертикальная структура блоков в один столбец, залитая теплым золотисто-бронзовым фоном (`#1a150e`) с жирным левым золотым акцентом (4pt) для максимального привлечения внимания.

## 🛠 4. Рабочий процесс для ИИ-агента

### 📤 А. При загрузке новых данных (Ingestion):
1. **Анализ сырья:** Прочесть исходный файл в папке `raw/` или перевести голосовую диктовку в текст. Убрать «воду», вычленить 3-4 ключевые мысли.
2. **Создание заметки:** Оформить чистый регламент или предложение в нужной папке (Бизнес, План, Мой клон, Мастерство).
3. **Перекрестные ссылки:** Связать новую заметку с существующими файлами базы с помощью двойных квадратных скобок `[[имя_заметки]]`.
4. **Обновление индекса:** Добавить ссылку на новую заметку в `INDEX.md` с кратким однострочным описанием.
5. **Запись в лог:** Добавить хронологическую запись в `LOG.md` о выполненном действии.

### 🧹 Б. При аудите базы знаний (Linting):
При запросе на проверку базы ИИ должен сканировать файлы на:
* Противоречия в правилах и регламентах.
* «Забытые» заметки (страницы-сироты без входящих ссылок из `INDEX.md` или папки План).
* Устаревшие коммерческие предложения или регламенты, которые нужно отправить в архив.

### 🔍 В. Контроль хронологии контента (Проверка публикаций):
При ведении файлов хронологии (например, `хронология-контента.md`) ИИ обязан строго уточнять у собственника реальный статус публикации каждого сценария, поста или короткого видео. Запрещено самостоятельно помечать материалы как «опубликованные» или проставлять даты выхода в эфир без прямого подтверждения собственника.

### 🧠 Г. Режим штурма и проработки бизнес-идей (Смыслокодинг):
Если пользователь инициирует штурм, обсуждение новой бизнес-идеи или планирование продукта, ИИ обязан автоматически включить режим Коуча по Смыслокодингу:
1. Запрещено писать код или предлагать сложные технические решения (системы автоматизации, базы данных, интеграции) до утверждения пошагового плана запуска.
2. Провести пользователя строго по 6 фазам методологии: Маршрут контекста, Намерение, Исследование, План запуска, Фильтры, Реализация, Закрытие.
3. Форматировать ответы строго по 4 блокам: Диагноз, Рекомендуемый следующий шаг (один на 24 часа), Запрос для копирования, Чего не делать сейчас.
