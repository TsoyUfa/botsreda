# ФРЕЙМ 1 — Обзор проекта

## Карточка: Суть проекта

**Назначение:** Единая точка входа на доску.

- **Проект:** Эксперт Сити — агентство новостроек, переход к экосистеме из 4 опор
- **Горизонт:** 6 мес (янв–июнь 2026) + квартальный фокус 12 недель
- **Цель:** Вывести агентство в топы по продажам новостроек
- **Ключевая метрика:** Рост продаж новостроек на **+50–100%**
- **Подход:** Последовательные продукты с синергией, не параллельный хаос

→ [МАРШРУТНАЯ КАРТА 2026 - Основной документ](Маршрутная%20карта%202026/МАРШРУТНАЯ%20КАРТА%202026%20-%20Основной%20документ.md)

---

## Карточка: Стратегический вывод (май 2026)

**Назначение:** Главный фокус — не распыляться.

- Сильная идея, но **высокий риск распыления** (4 направления одновременно)
- Не хватает **одного подтверждённого ядра** с цифрами и кейсом
- **Главная задача квартала:** доказать один рабочий контур, а не «разогнать всё»
- Формула успеха из Третьего Треста: **аудит → роли → KPI → пилот → контроль**
- Продавать через **ROI и кейс**, не через «у нас экосистема»

→ [СТРАТЕГИЧЕСКИЕ РЕКОМЕНДАЦИИ НА КВАРТАЛ](СТРАТЕГИЧЕСКИЕ%20РЕКОМЕНДАЦИИ%20НА%20КВАРТАЛ%20—%20ЭКСПЕРТ%20СИТИ.md)

## Карточка: Бренд и позиционирование

**Назначение:** Как представляем себя рынку.

- **Название (ребрендинг):** «Партнёры застройщиков»
- **Суть:** Партнёрская платформа между застройщиком, банком и клиентом
- **Обещание:** Рост агентского канала для девелопера + понятный путь сделки для клиента
- **Ценности:** Партнёрство, прозрачность, системность, репутация
- **Не подходим:** Тем, кто ждёт быстрый эффект без совместных усилий

## Карточка: Роль Антона (партнёр по развитию)

**Назначение:** Границы внешнего консультанта.

- Внешний партнёр по развитию — **не в линии управления**
- Цикл: диагностика → план → пилот → масштабирование
- Квартальная упаковка роли: **продукт + внедрение + метрики + измеримый эффект**
- Координация: директора, начальник департамента, РОПы/тимлиды
- Не делает: кадровые и дисциплинарные решения

# ФРЕЙМ 2 — Экосистема 3+1 продуктов

## Карточка: Карта экосистемы

**Назначение:** Как продукты усиливают друг друга.

```
Цифровой помощник ──► контент и данные ──► Telegram-канал
        ▲                                        │
        │                                        ▼
        └──── лиды и запросы ◄──── покупатели из канала
                        │
Консалтинг застройщиков ──► лучшие условия ──► агенты + канал + база
                        │
Академия ──► стандарты, скрипты, карточки ЖК ──► все продукты
```

---

## Карточка: Продукт 1 — Цифровой помощник

**Статус:** В работе | **Срок:** Мес 1–2 (2026)

- База новостроек с актуализацией для агентов
- MVP: поиск, фильтры, карточки ЖК, уведомления
- **KPI:** 50+ ЖК, 10+ активных агентов, −50% время поиска
- **Единица успеха:** частота использования, не факт «запуска»
- Фундамент для Telegram, академии и консалтинга

→ [ФАЗА 1](Маршрутная%20карта%202026/ФАЗА%201%20-%20Продукт%201%20Цифровой%20помощник.md) | [ПРОДУКТ 1](Маршрутная%20карта%202026/ПРОДУКТ%201%20-%20ЦИФРОВОЙ%20ПОМОЩНИК%20ДЛЯ%20АГЕНТА.md)

# ФРЕЙМ 3 — Квартальный фокус (12 недель)

## Карточка: Objective квартала

> **Создать доказанный внутренний контур:** Цифровой помощник + пилотные агенты + дисциплина использования + первые цифры «до/после».

**Не приоритет сейчас:**
- Агрессивное масштабирование Telegram
- Активные продажи консалтинга застройщикам
- Академия как отдельный большой проект

---

## Карточка: Key Results (квартал)

| # | KR | Цель |
|---|-----|------|
| 1 | Пилотная группа | 5–10 агентов с регулярным использованием |
| 2 | Рабочее ядро базы | 20–30 приоритетных объектов, обновление ≥1/нед |
| 3 | Время подбора | −30–50% внутри пилота |
| 4 | Кейсы | Минимум 3 оформленных кейса с пользой |
| 5 | Упаковка | 1 документ «до/после» для продаж и позиционирования |

---

## Карточка: Этап 1 — Недели 1–4 (собрать ядро)

**Цель:** Минимально рабочий контур, который реально используют.

- [ ] Определить пилотную группу 5–10 агентов
- [ ] Сузить базу до приоритетного ядра (топ-ЖК 2025)
- [ ] Зафиксировать стандарт обновления данных
- [ ] 5 сценариев: подбор, сравнение, аргументация, условия, подготовка к консультации
- [ ] Запустить еженедельный разбор с пилотом

**Результат:** рабочее ядро + первые пользователи + обратная связь

---

## Карточка: Этап 2 — Недели 5–8 (доказать полезность)

**Цель:** Влияние на скорость и качество, не «ощущение».

- [ ] Замерить время подбора «до/после»
- [ ] 5–10 типовых кейсов использования
- [ ] Убрать лишнее из MVP — только используемое
- [ ] Интегрировать материалы академии (карточки, скрипты, возражения)
- [ ] Начать внутренний кейс с цифрами

**Результат:** измеримые улучшения + чистый MVP

---

## Карточка: Этап 3 — Недели 9–12 (упаковать)

**Цель:** Пилот → аргумент для масштабирования.

- [ ] Краткий кейс «до/после»
- [ ] Решение: что масштабируем, что нет
- [ ] 10–15 единиц стартового контента для Telegram
- [ ] Короткая презентация ценности для застройщиков
- [ ] Внутренний стандарт работы агента с помощником

**Результат:** доказанный кейс + материалы для Q3

---

## Карточка: 5 живых метрик (еженедельно)

| Метрика | Зачем |
|---------|-------|
| Актуальных объектов в ядре | Качество базы |
| Активных агентов в пилоте | Adoption |
| Частота использования помощника | Реальный запуск |
| Среднее время подбора | ROI для агента |
| Кейсов «помог закрыть/ускорить» | Доказательство ценности |

## Карточка: 8 недель — 4 фазы

| Фаза | Недели | Фокус | Доступ к лидам |
|------|--------|-------|----------------|

| Практика | 5–6 | Воронка, живые клиенты | Под контролем |
| Интеграция | 7–8 | ИИ, метрики, защита кейса | Самостоятельно |

**Кодекс агента:** честность, ответственность, системность, ценность, дисциплина 85%

## Карточка: Trade-in и win-win

- Trade-in — **сервис с SLA**, не «обещание»
- Win-win: застройщик (лиды) + банк (ипотека) + клиент (условия) + агентство (канал)
- Co-brand мероприятия с застройщиками

→ [РЕБРЕНДИНГ - 05 Trade-in](Ребрейдинг/РЕБРЕНДИНГ%20-%2005%20Trade-in%20сервис%20—%20спецификация.md)
