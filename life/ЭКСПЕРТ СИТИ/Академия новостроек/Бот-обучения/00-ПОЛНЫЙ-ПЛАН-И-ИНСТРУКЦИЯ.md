# Telegram-бот «Академия новостроек» — полный план и инструкция

> Бот для обучения агентов по новостройкам: домашние задания, тесты, база знаний, мини-CRM, рассылки, аналитика ответов. Индивидуальный доступ каждому ученику.

---

## 0. Что строим (резюме одним абзацем)

Telegram-бот, в котором ученики (агенты) проходят обучение по новостройкам: получают теорию из базы знаний, выполняют домашние задания и тесты, видят свой прогресс. Ты (админ) создаёшь задания, видишь кто и что ответил, ведёшь базу учеников (CRM), рассылаешь сообщения — всё из админ-панели. У каждого ученика — личный доступ по пригласительному коду.

**Два входа в систему:**
- **Ученик** — общается с ботом в Telegram (получает задания, отвечает, читает базу знаний).
- **Админ (ты)** — управляет через встроенное админ-меню бота + (опционально) веб-панель FastAPI в браузере.

## 1. MVP Money Filter (проверка на деньги)

| Критерий | /5 | Комментарий |
|----------|-----|-------------|
| Путь к 500k+ | 4 | Обучение агентов = ядро услуг (диагностика→пилот→масштаб). Бот = удержание и масштабирование групп без роста твоего времени. |
| Скорость запуска | 4 | MVP реально собрать за 1-2 недели через Hermes. |
| Масштабируемость | 5 | Один бот на сотни учеников, группы, потоки. |
| Низкое сопротивление | 4 | Ученикам не нужно ставить приложение — всё в Telegram. |
| Win-win партнёров | 4 | РОП агентств получает прозрачный прогресс агентов; ты — продукт. |

- **Кто платит:** агентства недвижимости (РОП/директора) за обучение своих агентов, либо сами агенты.
- **За что платит:** за структурированное обучение с проверкой + аналитикой прогресса.
- **Почему сейчас:** обучение сейчас вручную → не масштабируется, нет прозрачности кто что усвоил.
- **Как проверить за 7 дней:** запустить 1 поток из 5-10 агентов на боте, прогнать 1 модуль + 1 тест, собрать обратную связь.
- **Что НЕ делаем в MVP:** видео-хостинг, оплату внутри бота, мобильное приложение, ИИ-проверку эссе, интеграцию с внешними CRM.

**Решение:** GO. MVP = авторизация + 1 модуль теории + 1 тест + просмотр ответов админом. Остальное — фазы.

- Роль определяется по `telegram_id`: если ID в таблице `admins` — права админа.

## 5. Детальное описание каждой механики

### 5.1 Доступ и авторизация

**Сценарий «Первый вход ученика»:**
1. Ученик открывает бота, жмёт `/start`.
2. Бот: «Введите пригласительный код».
3. Ученик вводит код (например `NOVOSTROY-7K2`).
4. Бот проверяет код в БД:
   - код верный и не использован / активен → создаёт/привязывает карточку ученика, сохраняет `telegram_id`, `username`, `full_name` (из профиля TG).
   - код неверный → «Код не найден, обратитесь к куратору».
   - код уже использован (одноразовый) → сообщение об ошибке.
5. Бот: «Добро пожаловать, [имя]! Вы в потоке „Поток 1". Ваш куратор: Антон.»
6. Открывается главное меню ученика.

**Сценарий «Повторный вход»:**
1. `/start` → бот видит `telegram_id` уже привязан → сразу главное меню.

**Сценарий «Отзыв доступа»:**
- Админ меняет статус ученика на «заблокирован» → бот при следующем действии отвечает «Доступ приостановлен, свяжитесь с куратором».

**Поля пригласительного кода:** `code` (уникальный), `group_id` (поток), `max_uses` (1 = одноразовый, N = многоразовый), `used_count`, `expires_at` (опц.), `active` (bool).

### 5.2 База знаний

**Структура:**
```
Модуль (напр. «Модуль 1: Рынок новостроек»)
  └── Урок 1.1 «Что такое первичный рынок»
       └── текст + фото + файл(опц.)
  └── Урок 1.2 «Долевое строительство: 214-ФЗ»
  └── ...
```

**Навигация ученика:**
1. Меню → «📚 База знаний».
2. Список модулей (inline-кнопки).
3. Выбор модуля → список уроков.
4. Выбор урока → текст урока + кнопки «◀️ Назад» / «Следующий урок ▶️».
5. После последнего урока модуля — кнопка «Пройти тест по модулю».

**Поиск:** команда `/search ипотека` → бот выдаёт список уроков, где встречается слово.

### 5.3 Задания и тесты

**Типы вопросов:**
- `single` — один правильный ответ (radio).
- `multiple` — несколько правильных (checkbox).
- `text` — открытый ответ (проверяет админ).

**Сценарий «Прохождение теста» (ученик):**
1. Меню → «📝 Тесты» → выбор теста (или из конца модуля).
2. Бот показывает вопрос 1: текст + варианты (inline-кнопки).
3. Ученик выбирает → бот сразу проверяет (для single/multiple) или принимает текст (для text).
4. Следующий вопрос. В конце: «Тест завершён. Ваш балл: 7/10. Открытые вопросы проверит куратор.»
5. Результат сохраняется в БД, виден админу.

**Конструктор теста (админ, через веб или команды бота):**
1. Админ: «Создать тест» → имя, привязка к модулю.
2. Добавить вопрос: текст, тип, варианты, правильные.
3. Сохранить.

**Ограничения (опц.):** лимит попыток, дедлайн по дате, таймер на прохождение.

### 5.4 Проверка и аналитика (админ)

**Лента проверки:**
1. Админ: «📋 Непроверенные ответы».
2. Бот показывает открытые ответы по очереди: ученик, вопрос, ответ.
3. Админ: ставит балл + пишет комментарий → уходит ученику.

**Сводка по тесту:**
- Сколько прошли, средний балл, % правильных по каждому вопросу (видно где ученики ошибаются).

**Прогресс ученика:** какие модули пройдены, какие тесты сданы, общий % выполнения.

### 5.5 CRM учеников

**Карточка ученика:**
- ФИО, @username, телефон (если дал), группа/поток, дата старта, статус.
- Прогресс по модулям.
- Средний балл.
- Заметки куратора.

**Действия админа:** смена статуса, смена группы, блокировка, добавление заметки, экспорт списка в CSV.

### 5.6 Рассылки

- **Всем / по группе / по статусу:** админ пишет текст → бот шлёт всем подходящим с задержкой (чтобы не словить лимит TG).
- **Индивидуально:** выбрать ученика → написать.
- **Отложенная:** задать дату/время → APScheduler отправит.
- **Отчёт:** X доставлено, Y ошибок (заблокировали бота и т.п.).

## 9. Критика: 10 причин провала + риски

| # | Риск | Критичность | Фикс |
|---|------|-------------|------|
