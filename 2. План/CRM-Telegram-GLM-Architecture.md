# 🧠 CRM-система на базе Telegram-бота + GLM 5.2 через Hermes

> **Дата:** 2026-07-03
> **Цель:** Полное техническое и логическое проектирование CRM внутри Telegram-бота, где хранятся знания, данные пользователей, вопросы. Ответы формируются ИИ на основе RAG (поиск по базе знаний) с использованием **GLM 5.2 через Hermes**.
> **Принцип:** Вся реализация и управление знаниями — через Hermes Agent. Основа — GLM 5.2 (Zhipu AI).

---

## 1. Контекст и исходное состояние

### Что уже есть в vault:
1. **«Главный бот»** (`Главный бот/`) — B2C-бот: лид-магниты, калькулятор ипотеки, 10-вопросный экспресс-тест, сбор лидов, drip-кампании. Стек: aiogram 3.x + sqlite3 (синхронный).
2. **Бот обучения** (`telegram-bot-obucheniya/`) — 7 блоков обучения, ДЗ с ИИ-проверкой, ЮKassa-платежи, бронирования, CRM-таблицы уже частично заложены (objects_jk, leads, deals, tasks).
3. **Hermes-бот** (`telegram-bot-hermes/`) — персональный бот Антона: голос → Gemini → GitHub/Obsidian.

### Что создаём:
Единая **CRM-система внутри Telegram-бота** с интеллектуальной обработкой вопросов пользователей на основе **базы знаний** и **GLM 5.2**.

## 3. Технологический стек

| Компонент | Технология | Обоснование |
|-----------|-----------|-------------|
| **Telegram-бот** | aiogram 3.x | Уже используется во всех ботах, async, FSM |
| **БД (операционная)** | SQLite → PostgreSQL (при росте) | Простота старта, миграция тривиальна |
| **Векторная БД (RAG)** | ChromaDB / FAISS (локально) | Хранение эмбеддингов знаний для семантического поиска |
| **AI-ядро** | **GLM 5.2** (Zhipu AI) через Hermes | Быстрый, русскоязычный, дешёвый токен |
| **Эмбеддинги** | GLM embedding API / BGE-M3 (локально) | Векторизация базы знаний |
| **Оркестратор** | **Hermes Agent** | Управление промптами, памятью, сценариями |
| **Голос → текст** | GLM 5.2 multimodal / Whisper | Распознавание голосовых вопросов |
| **Планировщик** | APScheduler | Drip-кампании, напоминания, задачи CRM |

### Почему GLM 5.2 через Hermes:
- **GLM 5.2** — сильная русскоязычная модель, низкая стоимость токена, быстрый отклик
- **Hermes** — провайдер-агностик: можно переключаться между GLM, Gemini, Claude без переделки кода
- **Память Hermes** — кросс-сессионная, накапливает профиль каждого пользователя
- **Навыки (skills)** — переиспользуемые сценарии (квалификация лида, проверка ДЗ, ответы по новостройкам)

### 5.2 Новые таблицы для CRM + RAG

```sql
-- ─── БАЗА ЗНАНИЙ (RAG-слой) ────────────────────────────────────

-- Чанки документов базы знаний (после разбиения на фрагменты)
CREATE TABLE IF NOT EXISTS knowledge_documents (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,              -- Название документа
    source_path TEXT,                 -- Путь к файлу в vault
    doc_type TEXT,                    -- 'methodology', 'faq', 'brandbook',
                                      -- 'lesson', 'script', 'objection', 'developer'
    category TEXT,                    -- 'ipoteka', 'sales', 'first_contact',
                                      -- 'financial', 'objections', 'brand'
    tags TEXT,                        -- JSON-массив тегов
    is_active BOOLEAN DEFAULT 1,
    chunk_count INTEGER DEFAULT 0,   -- Сколько чанков создано
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Векторные чанки (для семантического поиска)
-- Если ChromaDB: хранится там, здесь только ссылка
-- Если FAISS/локально: эмбеддинг в BLOB
CREATE TABLE IF NOT EXISTS knowledge_chunks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    document_id INTEGER NOT NULL,
    chunk_index INTEGER,              -- Порядковый номер чанка в документе
    content TEXT NOT NULL,            -- Текст фрагмента
    chunk_metadata TEXT,             -- JSON: страница, раздел, заголовок
    embedding_id TEXT,               -- ID в ChromaDB или NULL
    FOREIGN KEY (document_id) REFERENCES knowledge_documents (id)
);

-- ─── ВОПРОСЫ И ДИАЛОГИ (CRM-ядро) ──────────────────────────────

-- Каждое обращение пользователя (вопрос/сообщение)
CREATE TABLE IF NOT EXISTS user_questions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    question_text TEXT NOT NULL,      -- Оригинальный текст вопроса
    question_type TEXT,               -- 'b2c_purchase', 'b2b_consult',
                                      -- 'training', 'faq', 'support', 'unknown'
    intent_detected TEXT,            -- Определённый интент (через GLM)
    confidence_score REAL,           -- Уверенность классификатора (0.0–1.0)
    retrieved_chunk_ids TEXT,        -- JSON-массив ID чанков из RAG
    ai_response TEXT,                -- Сгенерированный ответ GLM
    response_model TEXT DEFAULT 'glm-5.2',
    tokens_used INTEGER DEFAULT 0,
    response_time_ms INTEGER,        -- Время генерации
    user_rating INTEGER,             -- Оценка ответа (1–5, от пользователя)
    is_escalated BOOLEAN DEFAULT 0,  -- Передан Антону (если ИИ не справился)
    status TEXT DEFAULT 'answered',  -- 'answered', 'escalated', 'resolved'
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users (user_id)
);

-- История диалогов (мультитёрн для контекста)
CREATE TABLE IF NOT EXISTS chat_sessions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    session_context TEXT,            -- JSON: накопленный контекст диалога
    started_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_message_at TIMESTAMP,
    is_active BOOLEAN DEFAULT 1,
    FOREIGN KEY (user_id) REFERENCES users (user_id)
);

-- ─── CRM: Лиды и сделки (расширение существующей схемы) ────────

-- Тегирование пользователей по сегменту
CREATE TABLE IF NOT EXISTS user_segments (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    segment TEXT NOT NULL,           -- 'b2c_buyer', 'agent', 'developer',
                                     -- 'student', 'partner'
    confidence REAL DEFAULT 1.0,     -- Как уверенно определён сегмент
    source TEXT,                     -- 'start_param', 'behavior', 'ai_classify'
    assigned_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users (user_id)
);

-- Воронка по работе с лидом (каждый шаг)
CREATE TABLE IF NOT EXISTS lead_funnel_stages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    stage TEXT NOT NULL,             -- 'new', 'qualified', 'calc_requested',
                                     -- 'meeting_scheduled', 'deal', 'closed'
    stage_metadata TEXT,             -- JSON: доп. данные этапа
    moved_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users (user_id)
);

-- Задачи (напоминания Антону и системе)
CREATE TABLE IF NOT EXISTS crm_tasks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,                 -- К кому привязана задача
    deal_id INTEGER,                 -- Связь со сделкой (опционально)
    task_type TEXT,                  -- 'follow_up', 'call', 'meeting',
                                     -- 'send_calc', 'check_homework'
    description TEXT NOT NULL,
    due_at TIMESTAMP,
    status TEXT DEFAULT 'pending',   -- 'pending', 'done', 'overdue', 'cancelled'
    created_by TEXT DEFAULT 'ai',    -- 'ai' (сгенерирована ИИ) или 'admin'
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMP
);

-- ─── АНАЛИТИКА И ОБУЧЕНИЕ СИСТЕМЫ ──────────────────────────────

-- Логирование всех AI-вызовов (контроль расходов)
CREATE TABLE IF NOT EXISTS ai_usage_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    question_id INTEGER,
    operation TEXT,                  -- 'classify', 'rag_search', 'generate'
    model TEXT DEFAULT 'glm-5.2',
    input_tokens INTEGER,
    output_tokens INTEGER,
    cost_rub REAL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Обратная связь (для дообучения и улучшения базы знаний)
CREATE TABLE IF NOT EXISTS feedback (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    question_id INTEGER,
    feedback_type TEXT,              -- 'thumbs_up', 'thumbs_down', 'correction'
    feedback_text TEXT,              -- Текст исправления (если есть)
    admin_reviewed BOOLEAN DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 6.3 RAG: построение базы знаний

**Источники знаний (документы vault):**

| Категория | Файлы | Объём |
|-----------|-------|-------|
| Методички | `data/lead_magnets/*.pdf`, `8. Методички/` | 8+ документов |
| Брендбук | `Брендбук агентства недвижимости СРЕДА.md` | ToV, принципы |
| Обучение | `6. обучения агентов/`, `life/ЭКСПЕРТ СИТИ/Академия/` | 7 блоков |
| Скрипты | `Серия reels про первый контакт/Лид-магниты/` | 8 скриптов |
| Застройщики | `9. Застройщики/` | Данные по ЖК Уфы |
| FAQ | Генерируется из `user_questions` со временем | Растущая база |

**Процесс индексации (через Hermes):**
1. Hermes читает все `.md` и `.pdf` файлы из указанных папок
2. Разбивает на чанки по 500–1000 токенов с перекрытием 100 токенов
3. Создаёт эмбеддинги каждого чанка (GLM embedding API)
4. Сохраняет в ChromaDB (коллекция `sreda_knowledge`)
5. Логирует в `knowledge_documents` и `knowledge_chunks`

**Переиндексация:** Cron-задача Hermes раз в сутки проверяет обновления в vault и добавляет новые/изменённые документы.

### 6.4 Системный промпт генерации ответа

```

1. Отвечай ТОЛЬКО на основе предоставленных знаний (контекст из RAG).
2. Если знаний недостаточно — честно скажи и предложи связаться с Антоном.
3. Тон: спокойно, по делу, без давления. Факты и цифры.

5. Если вопрос про ипотеку — указывай конкретные ставки и программы.

КОНТЕКСТ ПОЛЬЗОВАТЕЛЯ:

НАЙДЕННЫЕ ЗНАНИЯ (RAG):
{retrieved_chunks}

```

## 7. Логика маршрутизации (сценарии)

### 7.1 Матрица «вопрос → действие»

| Тип вопроса | Действие системы | Эскалация? |
|-------------|-----------------|-----------|
| **B2C: расчёт ипотеки** | AI объясняет схемы → предлагает калькулятор → собирает лид | Если нет ПВ → Антону |
| **B2C: выбор ЖК/района** | RAG по застройщикам → список подходящих | Всегда → лид |
| **B2C: ставка/программа** | FAQ из базы → конкретные цифры | Нет |
| **B2B: партнёрство** | AI объясняет формат → предлагает созвон | Да → лид + задача |
| **B2B: CRM/обучение** | AI объясняет → предлагает блок обучения | Если интерес → платный блок |
| **Обучение: ДЗ/урок** | AI даёт подсказку по материалу (без полного ответа) | Если 2 ошибки → куратору |
| **Обучение: тест** | AI оценивает свободный ответ (как сейчас Gemini) | Нет |
| **FAQ** | Прямой ответ из базы знаний | Нет |
| **Непонятный/сложный** | «Передаю Антону» → создаёт задачу | Да |
| **Жалоба/конфликт** | Эмпатичный ответ → немедленная эскалация | Да, приоритет |

### 7.2 Drip-кампании (автодогрев)

| Триггер | Задержка | Действие |
|---------|---------|----------|
| Скачал лид-магнит | +24ч | Приглашение на экспресс-тест |
| Прошёл тест (≥7/10) | +2ч | Предложение блока обучения |

## 8. Управление знаниями через Hermes

### 8.1 Hermes как «куратор базы знаний»

Hermes — не просто исполнитель, а **активный управляющий CRM-системой**:

1. **Индексация знаний** — Hermes автоматически читает новые файлы в vault, создаёт чанки, обновляет векторную базу
2. **Анализ unanswered questions** — раз в неделю Hermes просматривает вопросы без хорошего ответа и формирует новые FAQ-статьи
3. **Мониторинг качества** — Hermes анализирует 👎 обратную связь и предлагает улучшения
4. **Cron-задачи:**
   - Ежесуточно: переиндексация новых документов
   - Еженедельно: отчёт Антону по вопросам и конверсии
   - По триггеру: эскалация приоритетных лидов

### 8.2 Структура Hermes-навыка для CRM

