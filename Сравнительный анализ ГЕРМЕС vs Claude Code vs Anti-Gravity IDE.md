## Сравнительный анализ: ГЕРМЕС vs Claude Code vs Anti-Gravity IDE

Для четкого позиционирования инструментов при обучении или внедрении, их необходимо разделять по трем разным классам: **Автономный бизнес-интегратор** (Hermes), **Инженерный CLI-терминал** (Claude Code) и **Тяжелая ИИ-среда проектирования** (Anti-Gravity).

|**Критерий сравнения**|**HERMES (ГЕРМЕС)**|**Claude Code**|**Anti-Gravity IDE (Google)**|
|---|---|---|---|
|**Класс инструмента**|**Автономный ИИ-агент (Бизнес-автоматизатор)**|**Инженерный ИИ-агент (Консольный кодер)**|**Интерактивная среда разработки (IDE нового поколения)**|
|**Целевая аудитория**|Предприниматели, системные интеграторы, руководители, автоматизаторы|Разработчики, DevOps-инженеры, системные архитекторы|Программисты, инженеры данных, продуктовые разработчики|
|**Основной интерфейс**|Графическое приложение (Desktop GUI) + CLI|Исключительно Терминал командной строки (CLI)|Полноценный визуальный редактор (тяжелый GUI)|
|**Где выполняются вычисления?**|**Удаленный сервер** (подключение по SSH). Локальный Mac полностью разгружен.|Локальная машина (вызовы контекста идут через API Anthropic).|**Локальная машина** (требует огромных локальных ресурсов для удержания контекста).|
|**Фокус и суперсила**|Интеграция API, создание цепочек автоматизации (воркфлоу), работа по расписанию, сбор данных.|Глубокий рефакторинг кода, автономный поиск багов в репозитории, написание тестов, управление Git.|Написание архитектуры крупных систем «в четыре руки» с ИИ, интерактивное кодирование, сквозная индексация проектов.|
|**Автономность и память**|**Высокая.** Работает в фоне без участия пользователя, сохраняет навыки, обладает долговременной памятью.|**Средняя.** Работает автономно в рамках одной задачи, пока не решит её или не упрется в лимит.|**Низкая (Интерактивная).** Требует постоянного присутствия человека перед экраном, сессионная работа.|
|**Системный след на macOS**|Минимальный (кэш GUI, конфигурация SSH-ключей). Всё остальное живет на сервере.|Средний (кэш терминала, локальные конфигурации `.claude`).|**Максимальный** (гигабайты кэша индексов, локальные расширения, скрытые фоновые процессы обновления).|

## 3. Методические выводы для обучения (Takeaways)

Если вы упаковываете это в обучающий курс или регламент для команды, используйте следующие три правила позиционирования:

1. **Когда внедряем ГЕРМЕС:** Когда нужно связать несколько сервисов вместе, настроить регулярный парсинг данных, автоматизировать рутинные воркфлоу (маркетинг, отчетность, CRM-аналитика) и сделать так, чтобы система работала сама 24/7 на удаленном сервере, не отвлекая команду.
    
2. **Когда включаем Claude Code:** Когда перед техническим специалистом стоит жесткая инженерная задача в консоли — «найти почему падает сервер», «покрыть кодовую базу тестами», «быстро переписать старую функцию под новое API».
    
3. **Когда открываем Anti-Gravity / Классические IDE:** Когда мы садимся за проектирование систем, пишем архитектуру продукта с нуля, проводим ручной код-ревью или собираем сложные ИИ-конфигурации в реальном времени, требующие визуального контроля каждой строчки.
