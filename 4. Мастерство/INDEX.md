# 4. Мастерство

> Папка для чужого мастерства: концентрат методов лучших практиков, которые агент должен применять как экспертные линзы к задачам Антона.

## Зачем

`4. Мастерство` нужно, чтобы Cursor не отвечал «средним интернетом», а брал конкретную методологию под задачу:

- пост / оффер / презентация -> `copywriting/`;
- созвон, сделка, партнерство -> `negotiations/`;
- high-ticket консалтинг и продажи -> `sales/`;
- MVP, продукт, AI-автоматизация -> `product/` и `ai-automation/`;
- экономика, ROI, путь к 500k+ RUB/мес -> `financial-analysis/`;
- застройщики, новостройки, агентская сеть -> `real-estate-developers/`;
- критика решений и снижение важности -> `decision-making/`.

## Главное правило

Одна область = одна папка. Один автор, подход или методологическая линза = один файл.

В файл попадает не пересказ книги, а 1-3 метода:

- что это;
- когда применять;
- пошаговый алгоритм;
- какой результат должен получиться;
- какая информация нужна от Антона, если данных пока нет.

Если данных нет, ставить маркер: `[добавить инфу: что именно добавить]`.

## Роутинг для агента

Перед ответом выбери 1-2 релевантные папки из `4. Мастерство`, а не читай всё подряд.

Примеры:

- «Сделай оффер для застройщика» -> `sales/`, `copywriting/`, `real-estate-developers/`.
- «Проверь идею продукта» -> `product/`, `financial-analysis/`, `decision-making/`.
- «Подготовься к переговорам» -> `negotiations/`, `sales/`.
- «Автоматизируй CRM» -> `ai-automation/`, `product/`, `financial-analysis/`.

# 4. Мастерство

> Папка экспертных линз: как лучшие практики решают конкретные задачи. Это не AI-клон, не бизнес-досье и не архив книг, а сжатые методы, которые агент может применять к задачам Антона.

## Главное правило

- Один раздел = одна область мастерства.
- Один файл = один автор, школа или метод.
- Внутри файла нужны не пересказы, а 1-3 применимых метода: что это, как делать, когда применять.
- Если информации пока не хватает, ставить маркер `[добавить инфу]` и явно писать, какую информацию добавить.
- Перед использованием выбрать 1-3 релевантных файла, не загружать всю папку без необходимости.
