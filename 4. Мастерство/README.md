# Методология папки «4. Мастерство»

`4. Мастерство` - это не библиотека книг и не папка с красивыми саммари. Это набор рабочих методик, которые ИИ должен применять к задачам.

## Чем отличается от скилов

| Скил | 4. Мастерство |
|---|---|
| Повторяемый процесс: «сделай PDF», «создай презентацию», «обработай файл» | Методология предмета: «как продавать high-ticket», «как вести переговоры», «как считать ROI» |
| Может содержать скрипты и шаблоны | Содержит принципы, алгоритмы, критерии решений |
| Отвечает на вопрос «как выполнить операцию» | Отвечает на вопрос «как думал бы сильный эксперт» |

## Формат файла методологии

Каждый файл должен быть коротким и прикладным:

1. Кто автор / что за подход.
2. Когда применять.
3. Метод 1: суть, шаги, результат.
4. Метод 2: суть, шаги, результат.
5. Метод 3, если нужен.
6. Какие данные нужно добавить.
7. Связи с другими файлами.

## Ограничения

- Не хранить персональные данные клиентов.
- Не складывать полные книги и длинные транскрипты в рабочие файлы.
- Сырые материалы держать в `raw/`, а в методологию выносить только выжимку.
- Не плодить папки «на будущее», если нет задачи.
- Любой файл должен помогать принять решение, написать текст, продать, посчитать деньги или упростить процесс.

## Как пополнять

1. Взять книгу, выступление, курс, созвон или транскрипт.
2. Положить сырой материал в `raw/` или рядом с областью в `raw/`.
3. Попросить Cursor выделить только методы по формуле: что / как / когда / результат.
4. Перенести выжимку в файл автора или подхода.
5. Добавить в `INDEX.md` области одну строку: кто это и когда использовать.

## Главный фильтр Антона

Любая методология должна помогать одному из направлений:

- выйти на 500k+ RUB/мес;
- масштабировать консалтинг и AI-автоматизацию;
- усиливать недвижимость, новостройки, ипотеку, партнерские сети;
- снижать сложность и важность;
- создавать win-win для застройщиков, агентств, банков и клиентов.

# Методология папки «4. Мастерство»

## Зачем нужна папка

`4. Мастерство` нужно, чтобы агент не просто отвечал «из общих знаний», а применял конкретные методы сильных практиков к задачам Антона.

Эта папка отвечает на вопрос:

> Как бы эту задачу решал мастер в своей области?

## Чем отличается от `Мой клон`

`Мой клон` описывает, кто такой Антон: ценности, стиль, фокус, ограничения, способ принятия решений.

`4. Мастерство` описывает методы внешних экспертов: продажи, переговоры, копирайтинг, продукт, финансы.

Правильная цепочка:

```text
запрос -> Мой клон -> 4. Мастерство -> бизнес-контекст -> решение
```

## Формат файла метода

Каждый файл должен быть коротким и прикладным:

```text
# Автор или метод

## Кто это
3-5 строк, почему этот источник полезен.

## Метод 1
- Суть:
- Шаги:
- Когда применять:
- Как применить Антону:

## Метод 2
...

## Что добавить
- [добавить инфу] какие данные нужны для усиления файла.
```

## Что не класть

- Полные книги.
- Сырые транскрипты без выжимки.
- Биографии ради биографий.
- Абстрактные советы без алгоритма.
- Секреты, пароли, персональные данные клиентов.

## Как пополнять

1. Выбрать частую задачу.
2. Найти сильного автора / метод / выступление.
3. Вытащить 1-3 метода.
4. Описать каждый метод через «суть / шаги / когда применять / как применить Антону».
5. Сырые материалы складывать в `raw/`, а рабочий концентрат - в тематический раздел.
