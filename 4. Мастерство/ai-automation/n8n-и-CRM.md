# n8n и CRM

> Линза: n8n использовать для стабильных интеграций, а не как способ автоматизировать непонятный процесс.

## Когда применять

- Заявка -> CRM -> Telegram -> задача менеджеру.
- Ежедневный отчет по сделкам.
- Напоминания по follow-up.
- Сбор данных для диагностики.

## Метод 1. Автоматизация после 3 повторов

Шаги:

1. Выполнить процесс вручную.
2. Записать все шаги.
3. Повторить еще 2 раза.
4. Найти повторяемую часть.
5. Автоматизировать только ее.

Проверка:

- Есть стабильный вход?
- Есть понятный выход?
- Есть владелец процесса?
- Есть ошибка, которую можно безопасно обработать?
- Есть журнал действий?

## Метод 2. Минимальный сценарий CRM

Базовая цепочка:

1. Новая заявка.
2. Проверка обязательных полей.
3. Создание/обновление сделки.
4. Уведомление ответственного.
5. Контроль реакции.
6. Follow-up, если нет действия.
7. Лог результата.

## Метод 3. Безопасность и ПДн

Правила:

- Не отправлять телефоны, email, паспортные данные в LLM без маскирования.
- Для диагностики использовать обезличенные выгрузки.
- Не хранить токены в заметках.
- Доступы описывать как «где лежат», но не сами ключи.
