# Cursor и второй мозг

> Линза: Cursor в Obsidian - не только код, а рабочий интерфейс для стратегии, продукта, продаж и автоматизации.

## Когда применять

- Нужно структурировать заметки в рабочую систему.
- Нужно превратить идею в план, оффер или действие.
- Нужно создать контекст для будущих агентов.

## Метод 1. Контекст-инжиниринг

Порядок:

1. Короткий запрос.
2. Правильные файлы.
3. Метод из `4. Мастерство`.
4. Бизнес-контекст.
5. План или действие.
6. Ретроспектива / правило на будущее.

Главное: не грузить все заметки в один чат. Читать только релевантные файлы.

## Метод 2. Один чат - одна задача

Правило:

- один чат = один результат;
- большая задача = план-файл + фазы;
- контекст >60-70% = новый чат;
- после ошибки агента - правило в `Мой клон/feedback/` или `.cursor/rules/`.

## Метод 3. Роутинг по папкам

Пример:

- Кто я и как думаю -> `Мой клон/`.
- Как думать по предмету -> `4. Мастерство/`.
- Что делаем в бизнесе -> `Антон 2.0/`, `Застройщики/`, `Продукты/`.
- Что делать дальше -> планы / задачи / ретроспективы.
