# Настройка Qwen 3.7 как Claude Code в редакторе VS Code

> **Статус:** Регламент настройки ИИ-ассистента во Втором Мозге  
> **Цель:** Развернуть мощную локальную среду разработки без написания лишнего кода (no-code интерфейс) на базе китайской модели Qwen 3.7-max с минимальными затратами.

---

## 🎯 3 ключевые мысли из анализа технологии
1. **Экономия бюджета:** Подписка на план токенов (Token Plan) от Alibaba стоит от $20 в месяц и дает доступ к пулу передовых моделей (Qwen 3.7-max, DeepSeek V4, Kimi K2.6), что значительно выгоднее индивидуальной подписки на Claude Pro при ежедневном написании кода.
2. **Идеальный интерфейс:** Структура редактора VS Code полностью повторяет схему автономной экосистемы: слева проводник файлов проекта, справа - чат с ИИ-ассистентом для ввода запросов, по центру - окно генерации кода и его редактирования.
3. **Ловушки авторизации:** 90% ошибок подключения (код 401) связаны с неверным адресом API или тем, что пользователю не назначено рабочее место (Standard Seat) в консоли управления Alibaba Cloud.

---

## 💻 Архитектура интерфейса (No-code рабочее пространство)
Для работы в режиме визуального управления без ручного ввода команд мы собираем следующий интерфейс в редакторе VS Code:
* **Левая панель:** Встроенный проводник (дерево файлов проекта). Позволяет быстро ориентироваться в базе знаний или кодовой базе.
* **Центральная область:** Открытый файл с кодом. Здесь происходит автоматическая генерация, визуальное сравнение изменений и редактирование.
* **Правая панель:** Расширение **Qwen Code Companion** (устанавливается из маркетплейса VS Code). Это интерактивное окно чата, куда отправляются запросы и промпты.

## 🛠 Пошаговый алгоритм настройки

### Шаг 1. Подготовка окружения
1. Установите редактор **VS Code**.
2. Установите расширение **Qwen Code Companion** от разработчиков QwenLM через встроенную панель расширений.
3. Убедитесь, что на компьютере установлена среда выполнения Node.js версии 22 или новее.

### Шаг 2. Регистрация и покупка плана токенов (Token Plan)
1. Перейдите на международную консоль управления: `modelstudio.console.alibabacloud.com`.
2. Зарегистрируйте аккаунт и привяжите зарубежную банковскую карту (карты банков РФ не принимаются).
3. В разделе **ModelStudio** найдите вкладку **Token Plan** и выберите базовый тариф **Standard** ($20 в месяц).
4. **КРИТИЧЕСКИ ВАЖНО:** После оплаты перейдите в раздел подписок (My Subscriptions) и нажмите кнопку **Assign** (Назначить место) для своего аккаунта. Без этого шага статус будет равен `0/1` и система выдаст ошибку авторизации 401.

### Шаг 3. Получение ключей доступа
1. В консоли подписок нажмите фиолетовую кнопку **Go to Management Platform**.
2. В левом меню откройте раздел **Overview** (Обзор).
3. Скопируйте персональный ключ доступа **Plan-specific API Key** (формат ключа начинается на `sk-sp-...`).
4. Скопируйте правильный адрес сервера **Plan-specific Base URL** для OpenAI-совместимых инструментов:
   `https://token-plan.ap-southeast-1.maas.aliyuncs.com/compatible-mode/v1`

### Шаг 4. Конфигурация в VS Code
Для стабильного подключения рекомендуется настроить конфигурационный файл вручную, так как графический интерфейс расширения часто подставляет неверный адрес сервера.

1. Откройте или создайте файл настроек по пути `~/.qwen/settings.json`.
2. Вставьте в него следующую конфигурацию (замените `СЮДА-ТВОЙ-КЛЮЧ` на скопированный ключ `sk-sp-...`):

```json
{
  "env": {
    "TOKEN_PLAN_API_KEY": "sk-sp-СЮДА-ТВОЙ-КЛЮЧ"
  },
  "modelProviders": {
    "openai": [
      {
        "id": "qwen3.7-max",
        "name": "qwen3.7-max",
        "baseUrl": "https://token-plan.ap-southeast-1.maas.aliyuncs.com/compatible-mode/v1",
        "envKey": "TOKEN_PLAN_API_KEY",
        "generationConfig": {
          "extra_body": { "enable_thinking": true },
          "contextWindowSize": 262144
        }
      },
      {
        "id": "qwen3.6-plus",
        "name": "qwen3.6-plus",
        "baseUrl": "https://token-plan.ap-southeast-1.maas.aliyuncs.com/compatible-mode/v1",
        "envKey": "TOKEN_PLAN_API_KEY",
        "generationConfig": {
          "extra_body": { "enable_thinking": true },
          "contextWindowSize": 1000000
        }
      }
    ]
  },
  "security": {
    "auth": { "selectedType": "openai" }
  },
  "model": { "name": "qwen3.7-max" }
}
```

3. Перезапустите редактор VS Code.

## 🧹 Устранение типичных сбоев (Ошибка 401)
* **Проблема:** Ответ сервера `invalid access token`.  
  * **Решение:** Проверьте подписку в консоли. Убедитесь, что напротив пункта Standard Seat горит `1/1` (место назначено на вашу почту).
* **Проблема:** Ответ сервера `invalid_api_key`.  
  * **Решение:** Ключ формата `sk-sp-...` работает только с адресом `token-plan.ap-southeast-1...`. Проверьте, что в настройках не указан стандартный адрес DashScope.

## 🤖 Интеграция с ИИ-агентом Hermes Agent
Если вы хотите использовать модель Qwen 3.7-max в качестве главного мыслительного центра для своего ИИ-агента **Hermes Agent** от компании Nous Research, вы можете легко подключить ее благодаря стандартной архитектуре.

### Способ 1. Через встроенный мастер настройки
1. Откройте ваш терминал и запустите интерактивную команду:
   `hermes setup`
2. Выберите тип провайдера: `openai-compatible` (совместимый с OpenAI).
3. Введите адрес сервера (Base URL):
   `https://token-plan.ap-southeast-1.maas.aliyuncs.com/compatible-mode/v1`
4. Вставьте ваш ключ доступа: `sk-sp-...`.
5. Укажите имя используемой модели: `qwen3.7-max`.

### Способ 2. Ручная конфигурация через файлы настроек
Все системные параметры Hermes Agent хранятся в вашей домашней директории по пути `~/.hermes/`.

1. Откройте файл настроек окружения `~/.hermes/.env` и пропишите туда ваш секретный ключ:
   ```env
   CUSTOM_API_KEY=sk-sp-СЮДА-ТВОЙ-КЛЮЧ
   ```
2. Откройте конфигурационный файл `~/.hermes/config.yaml` и добавьте новый блок провайдера в раздел `providers`:
   ```yaml
   providers:
     custom-qwen:
       type: openai-compatible
       baseUrl: https://token-plan.ap-southeast-1.maas.aliyuncs.com/compatible-mode/v1
       apiKey: ${CUSTOM_API_KEY}
   ```
3. Проверьте корректность конфигурации в терминале командой:
   `hermes config check`

После этого ваш Hermes Agent начнет обрабатывать задачи, используя всю мощность Qwen 3.7-max по вашему тарифному плану токенов.
