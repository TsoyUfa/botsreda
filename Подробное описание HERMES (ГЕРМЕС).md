# База знаний: Экосистема ИИ-инструментов автоматизации и разработки

## 1. Подробное описание HERMES (ГЕРМЕС)

**HERMES (Hermes Agent / Hermes Desktop)** — это мультиплатформенная экосистема автономных ИИ-агентов нового поколения, спроектированная для долгосрочного планирования, автоматизации комплексных бизнес-процессов (воркфлоу) и интеграции разнородных сервисов через API.

### Ключевая концепция и архитектура

Философия Hermes строится на **разделении интерфейса и вычислений**. В отличие от классического софта, Hermes разделен на два компонента:

1. **Hermes Desktop (GUI) / CLI:** Легковесный пульт управления на локальном устройстве пользователя (например, Mac), который служит лишь для ввода задач, настройки связей и мониторинга процессов.
    
2. **Hermes Agent (Серверная часть):** Автономное ядро, которое разворачивается на удаленном сервере. Общение между пультом и ядром происходит по защищенному протоколу SSH.
    

Благодаря этой архитектуре, вся вычислительная нагрузка, работа с API, парсинг, выполнение скриптов и запуск фоновых задач происходят на удаленной машине. Локальный компьютер пользователя не расходует оперативную память, процессорное время и заряд батареи.

### Главные функциональные особенности Hermes:

- **Цикличность (Agent Loops):** Агент работает автономно циклами. Получив комплексную задачу, он сам разбивает её на подзадачи, планирует шаги (используя встроенные Канбан-доски), выполняет их, проверяет результат и при необходимости корректирует свои действия до победного финала.
    
- **Собственные навыки (Self-Improving Skills):** Агента можно обучать переиспользуемым навыкам (_skills_). Однажды созданный воркфлоу (например, «собрать аналитику по строящимся ЖК города, отфильтровать по цене за кв.м., упаковать в PDF и отправить в партнерский чат») сохраняется как кнопка быстрого запуска.
    
- **Многоуровневая память (Layered Memory):** Агент обладает долговременной памятью. Он способен накапливать контекст о вашем бизнесе, структуре ваших проектов, правилах коммуникации и предпочтениях, постоянно совершенствуя точность выполнения команд.
    
- **Фоновое исполнение по расписанию (Cron Jobs):** Задачи могут выполняться в фоновом режиме по часам, дням или триггерам. Если закрыть ноутбук или выключить локальное приложение, удаленный агент на сервере продолжит выполнять сценарии строго по графику.
