# Правила версионирования стратегий

> Любая стратегия в `01_strategy/` - это живой документ. При каждом изменении - фиксируем, что поменялось и почему. Старые версии не удаляются.

## Правило

При любом изменении стратегии:

1. **Обновить версию** в заголовке файла: `## Стратегия v2.0`
2. **Добавить запись** в секцию `## История изменений` (внизу файла)
3. **Никогда не удалять** предыдущие версии описания - только добавлять новые

## Формат записи истории изменений

```
### v2.0 - ДД.ММ.ГГГГ
- **Что изменилось:** [конкретные пункты]
- **Почему:** [на основании чего - анализ, созвон, инсайт]
- **Предыдущая версия:** v1.0
```

## Пример

```
## История изменений

### v3.1 - 12.06.2026
- **Что изменилось:** поменяли целевую аудиторию с «все агенты» на «агенты с выручкой 500к+/мес»
- **Почему:** анализ показал, что агенты ниже 500к не готовы платить за консалтинг
- **Предыдущая версия:** v3.0

### v3.0 - 01.05.2026
- **Что изменилось:** добавили продукт «цифровой помощник»
- **Почему:** запрос от Эксперт Сити
- **Предыдущая версия:** v2.0
```

## Где применять

- Все файлы в `1. Бизнес/01_strategy/`
- Стратегии по продуктам в `1. Бизнес/02_offers/`
- Контент-стратегии в блог/контент разделах
