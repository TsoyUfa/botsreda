# ПДн и безопасность

## Правила

- Не хранить секреты, токены и пароли в бизнес-заметках.
- Персональные данные клиентов не отправлять в LLM без обезличивания.
- Для CRM/БД использовать read-only доступ для аналитики.
- Email, телефон, ФИО маскировать при передаче агенту.
- Если проект собирает ПДн россиян, учитывать 152-ФЗ и хранение на российских серверах.

## Безопасная схема для AI

```text
CRM/БД -> API-прослойка -> обезличивание -> AI-анализ -> агрегированный отчёт
```

## Что можно хранить в `1. Бизнес`

- агрегированные цифры;
- описания сегментов;
- обезличенные кейсы;
- выводы и методологию;
- ссылки на источники без секретов.
