# Интеграция голосового ввода с Obsidian через Telegram

## Обзор
Система для автоматического создания заметок в Obsidian через голосовые сообщения в Telegram.

## Компоненты

### 1. Telegram-бот для быстрого ввода
Бот обрабатывает команды и голосовые сообщения:

#### Команды бота:
- `/note` - Создать заметку (автоопределение типа)
- `/business` - Бизнес-заметка (деньги, клиенты, офферы)
- `/plan` - Задача или план (дедлайны, проекты)
- `/clone` - Личная заметка (стиль, голос, принципы)
- `/mastery` - Методика или фреймворк
- `/help` - Справка по командам

#### Примеры использования:
```
/voice Сегодня встретился с застройщиком, он готов партнёрство по обучению агентов
```
→ Создаст заметку в `1. Бизнес/07_network/`

```
/voice Надо до конца недели сделать презентацию по AI-автоматизации для агентств
```
→ Создаст заметку в `2. План/01_projects/`

```
/voice Я всегда начинаю день с проверки дашборда и ответа на важные сообщения
```
→ Создаст заметку в `3. Мой клон/principles/`

### 2. Формат голосовых сообщений
Оптимальная структура голосового ввода:

```
[Тип контента] + [Суть] + [Детали]
```

Примеры:
- "Бизнес: Новое предложение по AI-консалтингу для застройщиков, цена 50к руб/месяц"
- "План: Запуск Telegram-бота для обучения агентов, дедлайн 1 июля"
- "Клон: Мой стиль общения - прямой, без воды, с конкретными цифрами"
- "Мастерство: Фреймворк квалификации клиентов по бюджету и срокам"

### 3. Обработка голосовых сообщений
Алгоритм обработки:

1. **Распознавание речи** → Текст
2. **Анализ контента** → Определение типа
3. **Создание заметки** → Правильная папка
4. **Добавление связей** → Связь с существующими
5. **Уведомление** → Подтверждение в Telegram

### 4. Автоматические уведомления
Система отправляет уведомления о созданных заметках:

```
✅ Заметка создана: "Новый партнер по обучению агентов"
📁 Папка: 1. Бизнес/07_network/partners-2026.md
🔗 Связи: [[Эксперт Сити]], [[застройщики]]
⏰ Время: 14:32
```

## Настройка

### 1. Создание Telegram-бота
1. Создать бота через @BotFather
2. Получить токен
3. Настроить webhook

### 2. Интеграция с Hermes
Добавить в конфигурацию Hermes:

```yaml
telegram:
  bot_token: "YOUR_BOT_TOKEN"
  webhook_url: "https://your-domain.com/telegram-webhook"
  voice_handler: "auto-structure-notes"
```

### 3. Cron-задачи для автоматизации
```bash
# Ежедневная синхронизация
0 9 * * * cd "/Users/anton_tsoy/Desktop/Обсидиан" && sync_telegram_notes.sh

# Часовая проверка новых сообщений
0 * * * * cd "/Users/anton_tsoy/Desktop/Обсидиан" && check_telegram_messages.sh
```

## Технические требования

### Обязательно:
- Telegram-бот с токеном
- Доступ к Obsidian vault
- Hermes Agent с навыком `auto-structure-notes`
- Настройка webhook для обработки сообщений

### Желательно:
- Сервер для 24/7 работы бота
- Система распознавания речи (можно встроенную в Telegram)
- Резервное копирование заметок

## Безопасность

### Защита данных:
- Все сообщения обрабатываются локально
- Никакие данные не передаются третьим лицам
- Голосовые сообщения удаляются после обработки

### Доступ:
- Бот доступен только авторизованным пользователям
- Возможность ограничения по Telegram ID
- Логирование всех действий
