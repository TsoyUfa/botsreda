# 🛠️ Техническое задание (ТЗ) на разработку Telegram-бота «Среда обучения»

**Цель бота:** Автоматизировать асинхронное обучение опытных агентов по недвижимости по программе «Методичка 2.0». Бот должен выдавать текстовые уроки, ссылки на опросники, принимать домашние задания (включая аудиосообщения) и отправлять их куратору на проверку.

---

## 🏗️ 1. Архитектура и стек
* **Платформа:** Telegram Bot API.
* **База данных:** SQLite / PostgreSQL (для хранения профилей пользователей, их прогресса и логов домашних заданий).
* **Панель куратора:** Упрощенная логика через закрытый Telegram-чат (Curator Chat), куда бот пересылает задания на модерацию с инлайн-кнопками.

---

## 📊 2. Модель базы данных (Рекомендуемая схема)

### Таблица `users` (Студенты)
* `user_id` (BIGINT, Primary Key) — Telegram ID пользователя.
* `username` (VARCHAR) — Никнейм в Telegram.
* `full_name` (VARCHAR) — Имя и фамилия, указанные при регистрации.
* `phone` (VARCHAR) — Номер телефона.
* `current_block` (INT, default: 1) — Номер текущего блока обучения (1..6).
* `current_lesson` (VARCHAR, default: "1.1") — Текущий активный урок (например, "1.1", "1.2").
* `status` (VARCHAR, default: "active") — Status: `active` (учится), `awaiting_review` (ждет проверки ДЗ), `completed` (завершил курс).
* `created_at` (TIMESTAMP) — Дата регистрации.

### Таблица `homeworks` (Домашние задания)
* `id` (INTEGER, Primary Key, Autoincrement)
* `user_id` (BIGINT, Foreign Key) — ID студента.
* `block_number` (INT) — Номер блока, по которому сдано задание.
* `file_type` (VARCHAR) — Тип файла: `voice` (голосовое для Блоков 1 и 4), `text` (текст для Блоков 2, 3, 5, 6).
* `file_id` (VARCHAR, Nullable) — Telegram File ID для голосовых сообщений.
* `status` (VARCHAR, default: "pending") — Статус проверки: `pending` (ожидает), `approved` (принято), `rejected` (отклонено).
* `curator_comment` (TEXT, Nullable) — Комментарий куратора при отклонении.
* `submitted_at` (TIMESTAMP) — Дата отправки.
* `reviewed_at` (TIMESTAMP) — Дата проверки.

## 🔄 3. Пользовательские сценарии и логика состояний (State Machine)

### Сценарий 1: Регистрация (Старт)
1. Пользователь отправляет команду `/start`.
2. Бот присылает приветствие и просит поделиться контактом (кнопка `[📱 Поделиться номером телефона]`).
3. После получения номера телефона бот запрашивает текстовым сообщением Имя и Фамилию.
4. Данные сохраняются в БД. Пользователю автоматически отправляется **Блок 1, Урок 1.1** (Сообщение 1 из контента Блока 1).

### Сценарий 2: Прохождение блоков и сдача ДЗ
1. Бот выдает Урок Х.1 с кнопкой перехода к Уроку Х.2.
2. В конце Урока Х.2 бот присылает описание ДЗ и переходит в состояние ожидания отправки сообщения (`state: awaiting_homework`).
3. Меню бота блокируется. Пользователь не может получить доступ к новым урокам, пока не отправит домашнее задание в нужном формате.
4. После отправки файла бот:
   * Сохраняет запись в таблицу `homeworks`.
   * Меняет статус пользователя в БД на `awaiting_review`.
   * Пересылает задание в Чат кураторов.
   * Отправляет пользователю сообщение: *«Ваше задание принято на проверку куратором. Ожидайте уведомления. Обычно проверка занимает не более 4 часов.»*

### Сценарий 3: Проверка куратором (Админ-панель в чате)
1. В чат кураторов приходит сообщение от бота:
   > **Новое ДЗ!**
   > **Студент:** [Имя Фамилия] (@username)
   > **Блок:** [Номер блока]
   > [Если это голосовое, бот прикрепляет аудиосообщение]
2. Под сообщением куратору доступны две инлайн-кнопки:
   * `[✅ Одобрить]`
   * `[❌ Отклонить]`
3. **При клике на `[✅ Одобрить]`:**
   * Статус домашнего задания меняется на `approved`.
   * У пользователя обновляется `current_block` (увеличивается на 1), `status` меняется на `active`.
   * Бот отправляет студенту уведомление: *«🎉 Ваше домашнее задание по Блоку Х одобрено! Вам открыт следующий блок обучения.»* и автоматически присылает Урок нового блока.
4. **При клике на `[❌ Отклонить]`:**
   * Бот переходит в состояние ожидания ввода причины отказа от куратора (в чате кураторов).
   * После того как куратор пишет причину (например, *"Тихий звук, перепиши без давления"*), статус ДЗ меняется на `rejected`, причина записывается в `curator_comment`.
   * Бот отправляет студенту уведомление: *«⚠️ Куратор проверил ваше ДЗ по Блоку Х и отклонил его.\n\n💬 Комментарий куратора: [Причина]\n\nПожалуйста, выполните задание заново и отправьте его в этот чат.»*
   * Пользователь остается в состоянии `awaiting_homework` для текущего блока.

## ⌨️ 4. Клавиатуры и интерфейс бота

### Постоянное Reply-меню (Кнопки внизу экрана)
*Выводится всегда после успешной регистрации.*
1. `🗺️ Мое обучение` — Выводит информацию о текущем блоке и уроке, дает ссылку на контент текущего урока.
2. `📝 Текущее ДЗ` — Повторно выводит условия домашнего задания для текущего блока.
3. `📊 Мой прогресс` — Показывает, сколько блоков пройдено (например: `Блок 1 [██░░░░] 16%`).
4. `❓ Поддержка` — Контакт куратора / РОПа для решения технических или методологических вопросов.

### Inline-кнопки под уроками
* `[✏️ Заполнить Точку А]` — Внешняя ссылка на Google Форму.
* `[✅ Я заполнил форму]` — Кнопка подтверждения действия.
* `[▶️ Перейти к уроку X.2]` — Кнопка навигации внутри одного блока.
* `[❓ Нужна помощь со скриптом]` — Вывод подсказки по ToV (для Блоков 1 и 4).

---

## ⚙️ 5. Системные требования к бот-билдеру
1. Бот должен поддерживать корректную обработку формата Voice (аудиосообщений Telegram), сохраняя `file_id` для пересылки кураторам.
2. Реализовать валидацию ввода: если в состоянии сдачи ДЗ пользователь присылает простой текст вместо аудио (для Блоков 1 и 4), бот должен вежливо напомнить: *«Для этого задания требуется записать голосовое сообщение. Пожалуйста, пришлите аудиозапись.»*
3. Сообщения должны выводиться с использованием разметки Markdown v2 для форматирования жирного текста, списков и цитат.
