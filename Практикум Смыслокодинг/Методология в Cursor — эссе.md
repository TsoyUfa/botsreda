# Смыслокодинг в Cursor — эссе и рабочая система

> Сжатая адаптация практикума Артемия Миллера под Obsidian + Cursor (Антон Цой). Источники: ключевые мысли и инструкции Дней 1–3.

---

## 1. Главная формула

**Смыслокодинг** = переложить экспертизу в продукт/решение через AI, а не «написать код ради кода».

**Контекст-инжиниринг** (2026) важнее длинных промтов:

```
короткий запрос → правильные файлы → метод → план → фаза → критика → результат → правило в базу
```

Для тебя: недвижимость, CRM, n8n, консалтинг, **500k+ RUB/мес**, путь наименьшего сопротивления.

## 4. Анатомия работы в Cursor

### 4.1. Пять слоёв контекста

1. Rules / карта навигации  
2. Профиль Антона (как думает)  
3. Mastery (метод под задачу)  
4. Бизнес-файлы (продукт, аудитория, CRM)  
5. Текущий чат + план-файл  

### 4.2. Один чат — одна задача

- Контекст >60–70% → новый чат или промт-перенос  
- Большая фича → **оркестратор** (план) + **исполнитель** (одна фаза)  
- Не параллелить агентов на одних файлах  

### 4.3. Цикл по задаче (6 шагов)

1. Человеческий запрос (голос OK)  
2. Уточнения до 95% ясности  
3. Research (рынок, аналоги) — файл, не просто чат  
4. План в `Планы/` с фазами `[ ]` / `[x]`  
5. Критика: «10 причин провала» + MVP-фильтр  
6. Реализация фазами → security review → деплой/демо

## 5. Ключевые промты (копируй в Cursor)

### Утренний фокус

```text
Прочитай: Антон 2.0 — профиль, Инбокс задач — май 2026, актуальный план в Планы/.
Цель: 500k+ RUB/мес. Дай 1 главный рычаг на сегодня и 1 действие на 24ч.
Путь наименьшего сопротивления. Формат: Контекст / Решение / Следующий шаг / Срок.
```

### План MVP (до кода)

```text
Не пиши код. Research: кто решал похожее, что продали, что упростить.
План в Антон 2.0/Планы/YYYY-MM-DD-название.md: фазы, критерий готовности, риск, деньги.
Обрежь всё лишнее для MVP за 7 дней.
```

### 10 причин провала

```text
Стресс-тест без утешений: 10 причин, почему не купят / не задеплоят / не принесут деньги.
Контекст: недвижимость, CRM, [продукт]. Конкретные правки по файлам.
```

### Security + ПДн

```text
Security review: .env, gitignore, ключи не во фронте, ПДн не в LLM без маски, 152-ФЗ если РФ.
Только то, что трогали в этой фазе.
```

### Перенос в соседний чат

```text
Напиши полный промт для нового чата: что сделано, что делать, какие файлы читать, что не трогать.
```

## 6. Антипаттерны

| Ошибка | Что делать |
|--------|------------|
| «Сделай всё» одним промтом | Разбить на фазы + план-файл |
| Транскрипт эфира в контекст | Только ключевые мысли / эссе |
| Идеальная инфраструктура до продажи | Сначала демо + 10 тёплых контактов |
| n8n/армия агентов с нуля | Ручной контур 3 раза → автоматизация |
| Хардкод KPI в markdown | LIVE + источник (CRM/таблица) |
| Cursor = только код | Vault = стратегия, КП, CRM-диагностика |
